      1. 데이터 현대화 및 AI Agent 기반 서비스 구축의 필요성

      KB는 급변하는 디지털 환경과 AI 기술의 발전 속도를 따라가기 위해 데이터 현대화와 AI Agent 기반 서비스 구축을 필수적인 전략으로 설정하고 있습니다. 기존의 레거시 시스템은 데이터 통합, 처리, 활용에 한계가 있으며, 특히 비정형 데이터(문서, 영상, 음성 등)의 관리와 활용이 부족한 상황입니다. 이에 따라 KB는 다음과 같은 문제점을 해결하고, 경쟁력을 강화하기 위해 데이터 현대화를 추진 중입니다.

      1-1. 기존 문제점

      • 데이터 사일로화: 부서 간 데이터 공유가 제한되어 있어, 통합 분석 및 AI 활용이 어려움.

      • 비정형 데이터 관리 부족: KB WiseNet, KB Drive 등에 산발적으로 저장된 문서, 영상, 음성 등의 데이터가 체계적으로 관리되지 않음.

      • AI 친화적 데이터 부재: AI가 이해하고 활용할 수 있는 구조화된 시맨틱 데이터가 부족.

      • 기술적 한계: 레거시 기반의 데이터 처리 방식이 AI Agent의 고속·실시간 분석 요구를 충족하지 못함.

      1-2. AI Agent 기반 서비스의 필요성

      • 고도화된 분석 능력: AI Agent는 복잡한 데이터를 자동으로 분석하고 인사이트를 도출할 수 있어, 기존 방식보다 훨씬 빠르고 정확한 의사결정을 가능하게 합니다.

      • 실시간 대응 능력: 실시간 스트리밍 데이터를 기반으로 AI Agent가 즉각적인 대응을 수행할 수 있습니다.

      • 업무 자동화 및 효율성 증대: AI Agent는 반복적이고 규칙적인 업무를 자동화하여 인력 효율성을 높이고, 고가치 업무에 집중할 수 있도록 지원합니다.

      • Agentic AI 및 생성형 AI 기술을 활용하여 차세대 업무 지원 체계를 구축할 수 있으며, 디지털 트윈, 디지털 워크플로우 등 미래형 업무 방식의 기반을 확보합니다.

      3-4. 솔루션의 기술독립과 출구전략 용이

      • 오픈테이블포맷(Iceberg, DeltaLake 등)을 사용하여 다양한 데이터 분석 솔루션(오픈소스, 상용, 클라우드 서비스 등)과 호환 및 대체가 가능합니다.

    • 데이터 인프라 아키텍처, AI 기반 서비스 아키텍처, 클라우드 및 보안 아키텍처 설명.첨부된 내용을 바탕으로, 데이터 인프라 아키텍처, AI 기반 서비스 아키텍처, 그리고 클라우드 및 보안 아키텍처를 중심으로 핵심적인 내용을 정리하고, 구현 방안을 자세히 설명드립니다.

      1. 데이터 인프라 아키텍처

      1.1 핵심 개념 및 구성 요소

      KB는 모던 데이터 플랫폼(Modern Data Platform)을 기반으로 데이터 레이크하우스(Data Lakehouse) 아키텍처를 도입하여, 다양한 유형의 데이터(정형, 비정형, 반정형)를 통합적으로 저장, 처리, 분석할 수 있는 인프라를 구축하고 있습니다. 이는 기존의 데이터 웨어하우스와 데이터 레이크의 장점을 결합한 구조로, Iceberg, Delta Lake 등 오픈테이블 포맷을 활용합니다.

      1.2 구현 방안

      • 레이크하우스 구축:

        • Iceberg 또는 Delta Lake를 기반으로 하여 오픈테이블 포맷을 적용.

        • 컴퓨팅/스토리지 분리 구조로 확장성과 유연성을 확보.

        • Bronze, Silver, Gold Layer 구조로 데이터 흐름을 관리:\n

          • Bronze Layer: 원천 데이터 원본 보존.

          • Silver Layer: 정제 및 품질 검증된 데이터.

          • Gold Layer: 비즈니스 로직에 맞춘 데이터 집합.

      • 데이터 카탈로그 고도화:

        • 비정형 데이터(문서, 영상, 음성 등)를 포함한 데이터 카탈로그를 확장.

        • DataHub 기반으로 커스터마이징하여 데이터 탐색 및 검색 기능 강화.

      • 데이터 표준 수집 체계:

        • 실시간/배치/API 방식으로 표준화된 데이터 수집.

        • 레가시 시스템 호환성 유지(한글 캐릭터셋, 데이터 형식 등).

        • 메타데이터 기반 파이프라인 운영.

      • 데이터 가상화 및 연결성 강화:

        • 분산된 데이터 소스를 논리적으로 통합.

        • API, MCP, 가상화 기술을 통해 Agent가 다양한 데이터 소스에 연결 가능.

      • 데이터 통합 및 거버넌스:

        • Data Mesh 도입으로 도메인 중심의 데이터 관리 강화.

        • 데이터 제품화(Data as a Product)를 통해 품질 및 활용성 증대.

        • DataOps 및 MLOps 체계를 구축하여 데이터 파이프라인 자동화.

      2. AI 기반 서비스 아키텍처

      2.1 핵심 개념 및 구성 요소

      KB는 Agentic AI와 생성형 AI 기술을 기반으로 리스크 Agent와 같은 AI 기반 서비스를 구축하고 있습니다. 이는 AI가 데이터를 기반으로 판단, 분석, 추천을 수행하는 체계로, 시맨틱 데이터(Semantic Data), AI 친화적 데이터 구조, Vector Store, ML/DL 분석 환경 등이 핵심입니다.

      2.2 구현 방안

      • 첨부 문서를 참고하여 \"AI 친화적 데이터 구조 구축\" 부분을 구체적인 구현 방안으로 수정하면 다음과 같습니다:

        AI 친화적 데이터 구조 구축

        • 시맨틱 데이터 기반의 의미와 관계 정리

          • 데이터를 의미와 관계로 연결한 지식그래프(Knowledge Graph)를 구축하여, 정형 및 비정형 데이터의 통합 활용을 지원.

          • GraphRAG(Graph Retrieval Augmented Generation) 기반으로 AI 모델의 환각(phantom) 현상을 완화하고, 보다 정확한 응답을 제공.

          • RDB와의 명확한 차별화를 통해 효율성 및 성능 향상을 도출.

          • 온톨로지(Ontology)를 기반으로 데이터의 의미적 구조를 정의하고, AI가 이해하고 추론할 수 있는 구조로 재구성.

        • AI 최적화 저장소 구현

          • Vector Store 활용:

            • 자연어, 이미지, 비정형 데이터를 벡터(Vector)로 변환하여 저장 및 검색.

            • AI 학습 및 추론 과정에서 빠른 유사도 검색 및 검색 속도 향상을 지원.

            • Pinecone, Milvus, Weaviate 등의 Vector DB 활용 예정.

          • In-Memory DB(메모리 기반 데이터 처리)

            • 빠른 응답이 필요한 Agent 작업 시, 실시간 처리를 위해 Redis 또는 Apache Ignite 등 메모리 기반 저장소를 활용.

            • 높은 처리 성능과 저지연(low latency) 환경을 제공.

          • NoSQL 및 Graph DB

            • 다양한 유형의 데이터를 유연하게 저장 및 관리할 수 있는 NoSQL(예: MongoDB, Cassandra) 및 Graph DB(예: Neo4j, TigerGraph)를 활용.

            • 비정형 데이터 및 복잡한 관계 데이터를 처리할 수 있도록 확장성 제공.

        • Feature Store 구축

          • 반복적으로 사용되는 AI 학습 및 추론 입력 데이터(Feature)를 일관성 있게 관리하고 제공.

          • ML Pipeline에서 사용되는 Feature를 저장 및 재사용하여 모델 개발 효율성 향상.

          • Tecton, ** Feast** 등 오픈소스 또는 커스터마이징된 솔루션 도입 검토.

        • AI Agent가 활용 가능한 구조 확보

          • Vector DB, In-Memory DB, Graph DB 등 다양한 저장소와 API 기반 연결(REST API, gRPC 등)으로 데이터를 실시간으로 제공.

          • WiseNet, KB Drive 내 비정형 데이터(문서, 영상, 음성 등)를 참조하기 위한 API 개발 및 연계.

          • ML 플랫폼과의 연동을 통해 리스크 관련 모델(예: 기업여신, 신용평가 등)에 사용되는 데이터를 Vector화하여 AI Agent에 직접 제공.

        • AI-Ready 데이터 제공 체계 마련

          • AI가 직접 활용할 수 있는 구조로 데이터를 전처리 및 변환.

          • 데이터 메타 정보(Metadata)와 의미 정보(Semantic 정보)를 함께 저장하여 AI가 데이터를 이해하고 활용할 수 있도록 지원.

          • SQL To Text, Text To SQL 등 AI가 데이터베이스를 이해하고 질의할 수 있도록 SQL Agent 구현.

      • 첨부한 문서를 바탕으로 \"ML/DL 분석 환경 구축\" 부분을 구체적인 구현 방안으로 재구성하면 다음과 같습니다:

        ML/DL 분석 환경 구축:

        1. 리스크 관련 모델 이관

        • 이관 대상 모델: 기업여신 모델, 신용평가 모델 등 리스크 Agent와 관련된 모델.

        • 이관 방식:\n

          • 레이크하우스 기반 데이터 제품화를 통해 기존 ML 모델을 데이터 제품으로 재구성.

          • ML 파이프라인 마이그레이션을 통해 기존 ML 플랫폼에서 현대화된 ML/DL 분석 인프라로 이관.

          • API 기반 배포: 이관된 모델은 예측 API로 배포되어 리스크 Agent 및 관련 어플리케이션에서 즉시 활용 가능.

        • 이관 범위:\n

          • ML 플랫폼 활용 데이터 (박데이터) 약 315TB 및 리스크 Agent 지원을 위한 복제 데이터 약 12TB 포함.

          • 리스크 Agent 파이프라인 구축을 위한 데이터 파이프라인 구성 및 운영 체계 도입.

        2. MLOps 체계 구축

        • AutoML 활용:\n

        • Model Registry:\n

          • 예측 API 서비스 지원을 통해 실시간 모델 적용 가능.

        • ML 파이프라인 자동화:\n

          • ML 분석 자원 생성 및 회수 기능으로 효율적 자원 관리.

          • 현행 분석가/프로젝트 모델 파이프라인 이관 지원을 통해 기존 작업을 현대화.

        3. ML EoS 대응 및 모델 지속 운영

        • ML 모델 업데이트 및 교체:\n

          • 리스크 Agent에 필요한 모델은 ML EoS 대응을 위해 지속적인 업데이트 및 교체.

          • ML 분석 솔루션 도입으로 기존 ML 모델이 지원되지 않는 기술 변화에 대응.

        • 리스크 Agent 내 ML 모델 개발 및 운영 지원:\n

          • ML/DL 분석 인프라 구축을 통해 리스크 Agent에서 필요한 모델 개발 및 운영 환경 제공.

          • MLOps 구현으로 모델의 자동 학습, 평가, 배포, 모니터링 기능 강화.

        4. ML 분석 공간 제공

        • 데이터 기반 ML/DL 분석 공간:\n

          • 레이크하우스와 데이터 제품화 기반으로 컴퓨팅 및 저장소를 제공.

          • 리스크 Agent에서 필요한 ML/DL 모델 개발 및 운영 환경 제공.

        • MLOps 구현:\n

          • 데이터 기반 모델 학습 및 실시간 예측 모델 API 배포 및 관리.

          • ML 분석 자동화를 통해 데이터 준비, 학습, 평가, 서빙, 모니터링 단계 자동화.

        • 리스크 Agent의 성능 향상: ML 모델의 지속적인 업데이트 및 자동화로 정확한 판단 능력 강화.

        • AI 경쟁력 확보: ML/DL 분석 환경 구축으로 데이터 기반 AI 경쟁력 확보.

        Agent 기반 서비스 확장

        1. 리스크 Agent 중심의 데이터 현대화 및 서비스 확장

        • 리스크 Agent 활용 데이터 이관:
          리스크 Agent가 필요한 내부 데이터(예: 기존 데이터 웨어하우스, 데이터 레이크 등)와 외부 데이터(공공기관 API, Bloomberg, IEA 등)를 기반으로 데이터 파이프라인을 구성하고, 이를 리스크 Agent에 연결하여 실시간/배치 방식으로 수집 및 처리합니다.

          • 정형 데이터는 데이터 가상화 기반 연결 방식을 우선 적용하여 데이터 이관 없이 가상 레이어에서 접근 가능하도록 구현.

          • 성능 및 처리 방식에 따라 필요 시 데이터 이관도 병행하여 최적의 처리 속도 확보.

        • 리스크 Agent의 데이터 파이프라인 개발:
          리스크 Agent가 활용할 데이터 파이프라인(약 600개 예상)을 개발하여, 기존 리스크 관련 모델(예: 기업여신, 신용평가 등)과 연결되도록 설계.

          • 기존 ML 플랫폼 내 리스크 관련 모델과 API 배포 파이프라인을 이관하여 리스크 Agent와 연계.

          • 파이프라인은 DataOps 및 MLOps 기반 자동화 프로세스를 도입하여 지속적인 데이터 흐름을 보장.

        2. AI-Ready 데이터 제공으로 Vector DB, Feature Store 등 빠른 서비스 지원

        • 시맨틱 데이터 구축:
          AI가 이해하고 활용할 수 있는 형태로 데이터를 정비.

          • 메타데이터뿐만 아니라 비즈니스 의미, 관계, 용어, 계산식 등도 체계적으로 정리하여 AI가 쉽게 파악할 수 있도록 구현.

          • 시맨틱 데이터를 기반으로 리스크 Agent의 성능을 향상시키고, AI가 데이터를 더 유연하게 활용할 수 있도록 지원.

        • Vector DB 및 Feature Store 지원:
          AI가 데이터를 빠르게 처리하고, 인사이트를 도출할 수 있도록 Vector DB와 Feature Store를 구축.

          • Vector DB는 이미지, 텍스트, 음성 등 다양한 형태의 비정형 데이터를 벡터 형태로 저장하여 AI 모델 학습 및 추론에 활용.

          • Feature Store는 공통된 데이터 피처를 저장 및 관리하여, 다양한 AI 모델에서 재사용 가능하게 설계.

        • AI-Ready 데이터 자동화 체계:

          • 데이터 카탈로그 고도화: KB WiseNet 내 산발적으로 저장된 문서, 영상, 음성 등 비정형 데이터를 탐색 및 활용할 수 있도록 데이터 카탈로그를 구축 및 확대.

          • 자동화된 요약, 분류, 키워드 추출: 비정형 데이터를 수집한 후, 자동 요약 및 분류 시스템을 통해 AI가 활용 가능한 구조로 전환.

          • 외부 데이터 자동 수집 및 연동: 외부 기관(공공기관, Bloomberg 등)의 데이터를 실시간 및 배치 방식으로 자동 수집 및 연동.

        3. 응답 시간 최적화를 위한 다양한 방식의 데이터 제공

        • 인메모리 기반의 빠른 조회:
          AI Agent가 빠르게 데이터를 조회할 수 있도록 인메모리 데이터베이스를 활용.

          • 실시간 성능이 중요한 리스크 판단, 금융 모델링 등에서는 인메모리 저장소를 사용하여 응답 시간을 단축.

        • 캐시 기반의 데이터 최적화:

          • 자주 조회되는 데이터는 캐시에 저장하여 반복적인 데이터 처리를 줄이고, 성능 향상.

          • 캐시 정책은 시간 기반, 사용 빈도 기반으로 설정하여 데이터의 최신성과 효율성을 동시에 확보.

        • OLTP DB 활용:

          • 트랜잭션 처리가 필요한 데이터는 OLTP 데이터베이스를 활용하여 빠른 처리와 정확한 데이터 관리를 보장.

          • 리스크 Agent가 실시간으로 데이터를 처리하고, 트랜잭션 기록을 남길 수 있도록 설계.

      • 데이터 기반 인사이트 서비스 구현 방안:

        • 내·외부 데이터 융합:
          내부 시스템(예: KB WiseNet, KB-Drive 등)에 저장된 비정형 데이터(문서, 영상, 음성 등)와 외부 공공기관 API, 금융 데이터, 시장 동향 데이터 등을 연계하여 데이터 레이크하우스를 기반으로 통합 저장소를 구축. 이를 통해 실시간 스트리밍, 배치 처리, API 연계를 통한 표준화된 데이터 수집 체계를 구현함.

        • 디지털 트윈 구현:

          • 내부 업무 프로세스와 고객 행동 패턴을 기반으로 디지털 트윈 모델을 개발하여 가상 환경에서의 시뮬레이션 및 최적화를 수행.

          • 예: 리스크 관리, 고객 서비스 프로세스, 금융 상품 개발 등에서 디지털 트윈을 활용하여 실제 운영 전의 시나리오 테스트 및 개선 방안 도출.

        • 디지털 워크플로우 구현:

          • 데이터 기반 자동화 워크플로우를 도입하여 업무 프로세스의 디지털화를 추진.

          • 예: AI Agent가 데이터를 기반으로 문서 자동 생성, 고객 이슈 분류, 리스크 예측 등을 수행하는 워크플로우 구축.

        • 신규 서비스 창출을 위한 인사이트 도출:

          • AI Agent와 생성형 AI 기술을 활용하여 데이터 분석 및 인사이트 도출.

          • 예: 고객 행동 패턴 분석을 통한 맞춤형 금융 상품 제안, 리스크 요인 분석을 통한 신용 평가 모델 개선, 시장 트렌드 분석을 통한 투자 전략 수립.

          • **비즈니스 도메인 정보(의미, 관계, 계산식 등)**를 반영한 시맨틱 데이터를 기반으로 AI가 이해하고 활용할 수 있도록 데이터를 정제.

        • 클라우드 기반 인프라 활용:

          • 클라우드 네이티브 아키텍처를 도입하여 확장성과 유연성을 확보.

          • 데이터 처리, 저장, 분석을 위한 클라우드 기반 레이크하우스 솔루션(CSP, Iceberg, DeltaLake 등)을 활용하여 다양한 유형의 데이터를 지원.

        • 데이터 기반 ML/DL 분석 환경 구축:

          • ML 모델 개발 및 배포를 위한 MLOps 체계를 구축.

          • 리스크 Agent, 고객 서비스 Agent 등 다양한 AI 어플리케이션에 활용할 수 있도록 ML 모델을 개발 및 이관.

        • 데이터 카탈로그 고도화:

          • 행내/외 비정형 데이터를 탐색 및 활용할 수 있도록 데이터 카탈로그를 고도화.

          • 오픈소스 기반의 DataHub를 커스터마이징하여 데이터 검색, 메타데이터 관리, 자동 분류 기능을 구현.

      3. 클라우드 및 보안 아키텍처

      3.1 핵심 개념 및 구성 요소

      KB는 클라우드 네이티브(Cloud Native) 기반의 아키텍처를 도입하여 확장성, 탄력성, 복원성, 유연성을 확보하고 있습니다. 이는 클라우드 기반의 인프라 운영, 보안, 규제 준수, 기술 전환 대응을 위한 체계입니다.

      3.2 구현 방안

      • 클라우드 인프라 최적화:

        • KB 표준 아키텍처 기반으로 클라우드 인프라 설계.

        • IaC(Infrastructure as Code) 기반으로 클라우드 자원 관리 (예: Terraform).

        • CSP(Cloud Service Provider)의 Managed Service(예: SageMaker Notebook) 활용.

        • 대량 트래픽 및 사용자 증가를 고려한 인프라 확장성 확보.

      • 보안 및 규제 준수 체계:

        • 제로 트러스트(Zero Trust) 기반의 접근 제어 강화.

        • 데이터 민감도 분류, 마스킹, 해시, 개인정보 보호 체계 구축.

        • 데이터 접근 로깅, 감사, 모니터링 체계 강화.

      • 클라우드 인프라 운영 체계:

        • KB One 클라우드 운영 가이드 준수.

        • Private Endpoint, Outbound 제어, IAM 권한 관리 등 보안 방안 적용.

        • CSP Object Storage Service를 활용한 데이터 저장 및 LifeCycle 정책 설계.

      • 기술 전환 대응 체계:

        • 오픈테이블 포맷(Iceberg, Delta Lake)을 활용하여 다양한 데이터 분석 솔루션(오픈소스, 상용, 클라우드) 호환 가능.

        • 기술독립성 확보 및 출구 전략(Exit Strategy) 용이.

      1. Agentic AI 기반의 서비스 아키텍처

      • Agentic AI는 AI 에이전트가 자율적으로 판단하고 실행하는 체계로, 기존의 규칙 기반 시스템에서 벗어나 인지, 추론, 실행 기능을 결합한 차세대 업무 지원 체계를 구축합니다.

      • 리스크 Agent는 AI 기반으로 리스크 판단 및 모니터링을 수행하며, 향후 서비스 확장성을 고려한 데이터 현대화 전략을 반영합니다.

      • 디지털 트윈 및 워크플로우 자동화를 기반으로 미래형 업무 방식을 구현하고, 내·외부 데이터를 융합하여 인사이트 기반의 신규 서비스를 창출합니다.

      2. 시맨틱 데이터와 지식 그래프

        • GraphRAG 기반으로 환각 문제를 해소하고, 더 정확한 응답을 제공.

      • 데이터 거버넌스: 데이터 수집, 가공, 분석, 페기의 생명 주기 수립 및 적용.

      • 접근 및 보안 관리: 데이터 접근 기록을 정기적으로 분석하고, 미활용 데이터를 자동으로 삭제.

      • 개인정보 및 민감 정보 보호: 마스킹, 해시 등을 활용한 보호 체계 구축.

      • 데이터 품질 관리: 오류 데이터 알림 제공 및 정합성 검증.

    1. 레이크하우스(Lakehouse)

    1.1. 기술 설명

    레이크하우스는 데이터 레이크와 데이터 웨어하우스의 장점을 결합한 새로운 데이터 아키텍처입니다. 데이터 레이크는 저비용의 저장 및 다양한 데이터 유형(정형, 비정형, 반정형)의 유연한 저장이 가능하며, 데이터 웨어하우스는 고성능 분석, ACID 트랜잭션, 데이터 품질 등을 지원합니다. 레이크하우스는 이 둘을 통합하여 단일 플랫폼에서 데이터 저장, 분석, ML/DL 모델 학습을 가능하게 합니다.

    1.2. RFP 기반 구현 방향

    • 컴퓨팅/스토리지 분리 아키텍처를 채택하여 확장성과 유연성을 확보합니다.

    • 오픈 테이블 포맷 (Iceberg, Delta Lake 등)을 기반으로 데이터의 호환성과 멀티엔진 지원을 강조합니다. 특히 Iceberg는 호환성이 높고, 멀티엔진(Apache Spark, Flink 등)에서 동작 가능하며, 데이터 이동 없이도 다양한 분석 엔진에서 활용 가능합니다.

    • ACID 트랜잭션 지원을 통해 데이터의 정합성을 유지하며, Upsert, Delete 등의 DML 작업이 가능합니다.

    • 데이터 레이어 구조 (브론즈, 실버, 골드 레이어)를 구현하여 데이터 품질과 재사용성을 높입니다.\n

      • 브론즈 레이어: 원천 데이터 보존

      • 실버 레이어: 데이터 클리닝, 가공

      • 골드 레이어: 분석, ML/DL 학습용 데이터

    1.3. 핵심 기술 요소

    • Iceberg 또는 Delta Lake 기반 오픈 테이블 포맷

    • 컴퓨팅/스토리지 분리 아키텍처

    • ACID 트랜잭션 지원

    • 브론즈-실버-골드 레이어 구조

    • 고성능 데이터 파이프라인 및 메타데이터 관리

    2. 시맨틱 데이터 (Semantic Data)

    2.1. 기술 설명

    시맨틱 데이터는 AI가 데이터를 '이해'할 수 있도록 의미와 관계를 명시적으로 구조화한 데이터 모델입니다. KB에서는 이를 기반으로 AI 친화적 데이터를 구축하여, 자연어, 이미지, 비정형 데이터 등을 벡터화하여 Vector DB에 저장하고, GraphRAG와 지식그래프를 통해 데이터의 의미적 맥락을 강화합니다. 이는 단순히 데이터의 정의나 속성을 기술하는 메타데이터를 넘어서, 계산식, 데이터 간의 관계, 의미적 연결, 비즈니스 용어의 정의 등을 포함한 데이터의 의미 구조를 재구성하는 것을 의미합니다.

    KB는 이를 위해 시맨틱 레이어를 구축하고, 정형 및 비정형 데이터를 데이터 레이크하우스와 Vector DB에 결합하여 AI가 다중 모달 데이터를 처리할 수 있도록 합니다. 특히, GraphRAG 기술을 통해 AI의 환각(Hallucination)을 해소하고, 더 정확한 추론과 응답을 제공하는 데 기여합니다. 이는 RDB와의 명확한 차별점을 확보하면서, 효율성과 성능 향상을 동시에 달성할 수 있는 구조를 구축합니다.

    KB는 이 기반 위에서 리스크 Agent를 포함한 Agentic AI를 구축하고, 시맨틱 데이터를 기반으로 한 AI 서빙을 위한 OLTP, Vector Search, ML/DL 도구, Data Store 등 다양한 인프라를 클라우드 기반으로 최적화했습니다. 이를 통해 AI 서빙의 지연 최소화, 데이터 접근성 강화, ML 파이프라인의 일관성을 보장하며, AI 기반 업무 지원 체계를 구축합니다.

    또한, DataOps와 MLOps 체계를 통해 데이터 품질, 정합성, 거버넌스를 강화하고, Data as a Product 개념을 도입하여 데이터의 재사용성 및 가치창출을 극대화합니다. 이 모든 과정은 KB가 자체적으로 구축한 DataHub 기반의 데이터 카탈로그와 외부 비정형 데이터 수집 체계를 통해 가능합니다. 이로 인해 KB는 AI Agent 성능 향상과 AI 기반 비즈니스 적용의 가속화를 기대할 수 있습니다.

    2.2. RFP 기반 구현 방향

    • 시맨틱 데이터 저장소 구축:
      KB는 AI가 이해하고 활용할 수 있는 구조로 데이터를 정비하기 위해 시맨틱 데이터 저장소를 구축합니다. 이 저장소는 단순한 메타데이터를 넘어, 데이터의 비즈니스 의미, 관계, 용어, 계산식 등을 포함하여 AI Agent가 데이터를 자연스럽게 해석하고 활용할 수 있도록 지원합니다.
      이에 따라, GraphRAG 기술을 활용해 데이터 간의 환각 문제를 해소하고, 더 정확한 응답을 제공하는 체계를 마련하고 있습니다. 또한, Iceberg와 같은 오픈 테이블 포맷을 기반으로 데이터의 자율성과 호환성을 확보하여 다양한 분석 도구와의 연계를 가능하게 합니다.

    • 지식 그래프 기반의 관계 분석:
      KB는 지식 그래프를 활용한 데이터 관계 분석을 통해 데이터 간의 의미적 연결성을 시각화 및 활용하고 있습니다.
      이는 RDB와의 명확한 차별화를 이끌어내며, 특히 정형과 비정형 데이터의 연계를 강화하여 AI가 다양한 데이터 소스를 기반으로 인사이트를 도출하는 데 기여합니다.
      온톨로지 기반의 지식 그래프는 데이터의 의미를 체계적으로 표현하며, AI의 추론 능력과 정확도를 향상시키는 핵심 기반으로 활용됩니다.

    • 데이터 카탈로그 고도화:
      KB는 데이터 카탈로그를 비정형 데이터까지 확장하여, 데이터 탐색 및 활용의 편의성을 강화하고 있습니다.
      특히, WiseNet, KB 드라이브 등 산발적으로 저장된 비정형 데이터를 수집하고, 이를 자동화된 요약, 키워드 분류, 관계 맵핑을 통해 시맨틱 데이터로 전환하여 데이터 카탈로그에 반영합니다.
      이는 AI Agent가 데이터를 빠르게 탐색하고 활용할 수 있는 기반을 제공하며, AI 친화적인 데이터 환경 구축에 기여합니다.

    • AI 친화적 데이터 구조 지원:
      KB는 AI가 자연어, 이미지, 음성 등 다양한 형태의 데이터를 벡터로 변환하여 저장하고 활용할 수 있도록 Vector DB를 도입하고 있습니다.
      또한, Feature Store를 통해 반복적으로 사용되는 AI 학습 및 추론에 필요한 입력 데이터를 일관성 있게 제공하는 체계를 구축하고 있습니다.
      이를 통해, AI Agent가 저지연으로 데이터에 접근하고, 고품질의 인사이트를 제공할 수 있는 환경이 조성됩니다.

    • **네오알리의 차별화된 전략: KB는 네오알리의 지식 그래프 기반의 시맨틱 데이터 플랫폼을 도입하여, AI Agent의 데이터 이해력과 추론 능력을 극대화하고 있습니다.
      네오알리는 데이터의 의미와 관계를 체계적으로 추출·연계하여, AI가 단순히 패턴을 학습하는 것을 넘어 비즈니스 맥락을 이해하고 인사이트를 도출할 수 있도록 지원합니다.
      특히, KB는 데이터 레이크하우스와 연계된 네오알리 플랫폼을 통해, 정형/비정형 데이터의 통합 분석과 AI Agent의 실시간 판단 능력 강화를 동시에 달성하고 있습니다.
      이는 KB가 차세대 AI 기반 업무 지원 체계를 구축하고, 디지털 트윈 및 워크플로우 혁신을 이끌어내는 데 중요한 역할을 하고 있습니다.

    시맨틱 데이터 구현을 위한 핵심 기술 요소

    1. 시맨틱 레이어 구축

    • 의미 기반 데이터 재구성:
      데이터를 단순한 값이 아닌 의미와 관계를 갖는 메타 정보로 재구성하여 AI가 이해하기 쉬운 구조로 전환합니다.
      예: \"예금금리\"는 단순한 숫자가 아닌, \"은행의 자금 운용 전략\", \"시장 금리\", \"고객 수요\" 등과 연관된 의미를 갖습니다.

    • 데이터 마트 기반 시맨틱 구조화:
      도메인별 데이터 마트를 생성하고, 마트 내 메타 정보를 보완하여 시맨틱 레이어에서 활용합니다.
      예: \"리스크 데이터 마트\" 내에서 SQL 텍스트를 기반으로 Join 정보를 추출하여 시맨틱 맵핑을 확장합니다.

    • SQL Agent와 Text-to-SQL 확장:
      SQL 텍스트를 기반으로 AI가 데이터 패브릭 구조로 확장하여, 단일 데이터 소스가 아닌 다중 데이터 소스를 통합적으로 활용할 수 있도록 지원합니다.

    2. 지식 그래프 기반 관계 분석 (Ontology-based RAG)

    • 도메인 온톨로지 구축:
      NEOALI는 금융 및 규제 분야에 특화된 온톨로지를 구축합니다.

      • 엔티티 추출: \"은행\", \"정책\", \"금융 상품\", \"규제 기관\" 등

      • 관계 모델링: \"은행 → 규제 기관\", \"정책 → 유효 일자\", \"금융 상품 → 조건\" 등

      • 시간적 속성: 정책의 유효 기간, 상품 변경 이력 등

    • 엔티티 관계 맵핑 (Entity Relationship Mapping):
      문서 내에서 명확히 구분되지 않은 엔티티 간 혼동을 방지합니다.
      예: \"김병은\"과 \"김명준\"이 같은 이름일 수 있으나, 상황에 따라 다른 인물일 수 있음.
      NEOALI는 온톨로지를 통해 엔티티를 구분하고, 정확한 관계를 맵핑합니다.

    • 인과 관계 분석 (Causality and Implication):
      시맨틱 데이터를 기반으로 인과 관계를 추론합니다.
      예: \"자본비율 하락\"은 시장 손실, 자산 평가 감소, 규제 변경 등 여러 요인과 연관될 수 있습니다.
      NEOALI는 이러한 관계를 추적하고, 원인을 명확히 설명하는 응답을 제공합니다.

    3. 데이터 카탈로그에 시맨틱 정보 포함

    • AI와 비즈니스 사용자를 위한 메타데이터 탐색 환경:
      데이터 카탈로그에 시맨틱 정보를 포함하여, 사용자가 자연어로 데이터를 검색하고 탐색할 수 있도록 지원합니다.
      예: \"리스크 관련 데이터\"를 검색하면, 관련된 규제, 상품, 이력 등을 시맨틱 맵핑 기반으로 제공.

    • Vector DB 활용:
      자연어, 이미지, 비정형 데이터를 벡터로 변환하여 저장 및 검색을 지원합니다.

      • Vector DB 기반 데이터 저장: 시맨틱 정보를 벡터화하여 저장

      • GraphRAG 기반 응답: GraphRAG를 통해 AI가 시맨틱 정보를 활용하여 환각 없는 응답을 제공

    • Feature Store 기반 반복 데이터 지원:
      AI 학습 및 추론에 반복적으로 사용되는 데이터를 일관된 방식으로 제공합니다.
      예: 고객 행동 패턴, 거래 이력, 리스크 패턴 등

    4. NEOALI의 독창적 기술 적용

    • NEOALI Knowledge Parser:

      • 시맨틱 파싱 기술: 문서의 구조적 요소(장, 절, 조, 항 등)를 정확히 인식하고, 의미를 추출합니다.

      • OCR 기반 스캔 문서 처리: 금융 문서에 최적화된 OCR 기술로, 정확한 텍스트 추출과 시맨틱 정보 인식을 지원합니다.

      • 다양한 포맷 지원: Word, PDF, EML, PPTX, XLSX 등 25가지 이상의 포맷을 지원하며, 시맨틱 정보를 일관되게 추출합니다.

    • Hybrid RAG 파이프라인:

      • Sparse Retrieval (BM25): 정확한 키워드 매칭을 통해 정책 번호, 계좌 ID, 규제 코드 등을 빠르게 검색합니다.

      • Dense Retrieval (Vector Embeddings): 의미 기반으로 관련 정보를 추출합니다.
        예: \"유동성 요구사항\"과 \"현금 예약\" 문서가 의미적으로 관련되어 있을 경우, 정확히 연결합니다.

      • Intelligent Re-Ranking: Cross-encoder 모델로 관련성을 재검토하여, 가장 정확한 정보만 제공합니다.

      • LLM Verification Layer: 최종적으로 AI가 추출한 정보가 사용자의 질문에 정확히 답하는지 확인합니다.

    결론

    NEOALI는 시맨틱 데이터를 구현하기 위해 의미 기반 데이터 재구성, 지식 그래프 기반 관계 분석, Vector DB 및 Feature Store 활용, Hybrid RAG 파이프라인, NEOALI Knowledge Parser 등의 독창적인 기술을 결합합니다.
    이러한 기술은 AI가 정형 및 비정형 데이터를 이해하고, 규제, 리스크, 고객 행동 등 다양한 도메인에서 정확한 인사이트를 제공할 수 있도록 지원합니다.

    3. ML/DL 분석 환경 및 MLOps

    3.1. 기술 설명

ML/DL 분석 환경은 머신러닝과 딥러닝 모델의 학습, 평가, 배포, 모니터링을 위한 컴퓨팅 및 저장 인프라를 의미합니다. MLOps는 ML/DL 모델의 개발, 배포, 운영, 모니터링을 자동화하고 표준화하는 운영 체계입니다.

본 프로젝트에서는 레이크하우스와 데이터 제품을 기반으로 ML/DL 분석 공간을 구축하여, 데이터 기반의 모델 학습 및 실시간 예측 모델 API 배포 및 관리의 MLOps를 구현합니다. 이는 데이터 추출, 분석, 준비, 학습, 평가, 검증, 서빙, 모니터링 등 ML 모델 운영의 자동화를 지원하며, 분석가 작성 코드 이력 관리를 위한 repository를 구축하고, Feature 확산 및 공유 기능을 제공합니다. 또한, 모델 품질 평가, 재학습, 예측 API 서비스 지원, 자동화된 ML 파이프라인 운영 관리, ML 분석 자원 생성 및 회수 기능을 포함합니다. 리스크 Agent 내 ML 모델 파이프라인 개발 지원도 함께 수행됩니다.

ML/DL 분석 인프라 및 솔루션은 KubeFlow, 레이크하우스 분석 솔루션, CSP 서비스 등을 도입하여 구현하며, ML/DL 분석 환경은 다음과 같은 주요 기능을 포함합니다:

  • 자동화된 ML 파이프라인 운영: 분석가/프로젝트 모델 파이프라인 이관 지원을 통해 운영 효율성을 극대화합니다.

  • 모델 품질 평가 및 재학습: 모델의 품질을 지속적으로 평가하고, 재학습을 통해 최적화된 성능을 유지합니다.

  • 예측 API 서비스 지원: API 기반의 모델 서빙을 통해 실시간 예측 서비스를 제공합니다.

  • 자원 관리: ML 분석 자원 생성 및 회수 기능을 통해 효율적인 자원 활용이 가능합니다.

  • 리스크 Agent 연계: 리스크 Agent에서 필요한 ML/DL 모델 개발 및 운영 환경을 제공합니다.

이러한 기능은 데이터 추출부터 모델 배포, 모니터링까지의 전 과정을 자동화하며, 보다 높은 수준의 데이터 기반 의사결정을 가능하게 합니다. 이를 통해 KB 내부의 ML/DL 분석 역량을 강화하고, 리스크 관리, 고객 맞춤 서비스, 자동화된 O&M 등 다양한 분야에서의 활용 가능성을 확대할 수 있습니다.

  • 3.2. ML/DL 분석환경 개발 및 MLOps 구현 방향

    1. ML/DL 분석 공간의 구축

    • 레이크하우스와 데이터 제품(Data Product) 기반의 분석 환경을 구축하여, 정형 및 비정형 데이터를 하나의 플랫폼에서 통합적으로 처리하고 분석할 수 있도록 합니다.

    • 레이크하우스는 데이터 웨어하우스와 데이터 레이크의 장점을 결합한 구조로, 데이터 저장, 처리, 분석, 모델링 등 다양한 작업을 지원합니다.

    • **데이터 제품화(Data as a Product)**를 통해 데이터의 신뢰성, 표준화, 접근성 및 재사용성을 강화하고, 각 도메인별로 데이터를 제품 형태로 제공하여 분석 및 활용이 용이하도록 합니다.

    2. MLOps 체계 구축

    • ML/DL 모델 학습 및 실시간 예측 모델 API 배포 및 관리를 위한 MLOps 체계를 구축합니다.

    • MLOps는 모델의 전 생명주기(데이터 준비 → 모델 개발 → 학습 → 배포 → 모니터링)를 자동화하고, 팀 간 협업과 반복 가능한 워크플로우를 지원합니다.

    • ML/DL 분석 인프라 및 솔루션 도입:\n

      • 데이터 추출, 분석, 준비, 학습, 평가, 검증, 서빙, 모니터링 등 ML 모델 운영을 자동화합니다.

      • 분석가 작성 코드 이력 관리를 위한 repository 구축으로 모델의 기록 추적과 재사용성을 확보합니다.

      • Feature 확산 및 공유, 모델 품질 평가, 재학습, 예측 API 서비스 지원 등을 포함하여 모델의 지속적 개선과 운영을 지원합니다.

      • 자동화된 ML 파이프라인 운영 관리를 통해 반복 작업을 효율적으로 처리하고, 리스크 Agent에서 필요한 모델 파이프라인을 개발 및 운영합니다.

    3. 오픈소스 기반 MLOps 도구 활용

    • KubeFlow, MLFlow, Airflow 등의 오픈소스 기반 MLOps 도구를 활용하여 모델의 전 생명주기 관리를 가능하게 합니다.\n

      • KubeFlow: Kubernetes 기반의 ML 워크플로우 자동화 플랫폼으로, 모델 학습, 테스트, 배포 등을 관리합니다.

      • MLFlow: 모델 실험, 패키징, 배포, 추적을 위한 포괄적인 플랫폼으로, 모델 성능 분석과 재사용성을 강화합니다.

      • Airflow: 데이터 파이프라인의 스케줄링 및 모니터링을 지원하여, 데이터 처리 및 모델 학습 프로세스를 자동화합니다.

    4. 리스크 Agent에서의 ML/DL 모델 운영 지원

    • 리스크 Agent에서 필요한 ML/DL 모델 개발 및 운영 환경을 지원합니다.

    • 리스크 관련 모델 (예: 기업여신, 신용평가 등)을 기반으로, ML 플랫폼 내 리스크 파이프라인을 구성하고, API 배포 및 모델 운영을 지원합니다.

    • 리스크 Agent의 데이터 및 파이프라인 마이그레이션을 통해 기존 시스템에서 ML/DL 기반 분석 플랫폼으로 전환하며, 검증 후 확산합니다.

    5. 데이터 품질 및 거버넌스 강화

    • 데이터 품질 측정 기반의 데이터 제품화(Data as a Product)를 통해 데이터의 신뢰성과 활용성을 높입니다.

    • 정형/비정형 데이터 관리체계 및 메타 정보 관리를 확대하여, 데이터 카탈로그의 검색 및 활용을 강화합니다.

    • 데이터 민감도에 따른 분류 및 접근제어, 개인정보 보호 체계(마스킹, 해시 등), 데이터 접근 로깅 및 규제 준수 모니터링 등을 통해 데이터 보안을 강화합니다.

    6. 클라우드 기반 인프라 및 확장성

    • 클라우드 네이티브 설계 및 구축으로, 확장성, 탄력성, 복원성을 갖춘 인프라를 구축합니다.

    • 클라우드 기반의 최신 기술과 서비스를 적극 활용하며, 향후 기술 전환을 고려한 구조로 설계합니다.

    • 인프라 구성은 IaC(Infrastructure as Code) 기반으로 구현되어야 하며, 클라우드 및 시스템 자원을 코드 기반으로 정의, 변경, 배포, 관리할 수 있어야 합니다.

    7. 장애 대응 및 복구 체계

    • 데이터의 버전 관리(Time Travel) 및 백업/복구 체계를 구축하여, 데이터 손실 시 신속한 복구가 가능하도록 합니다.

    • 장애 시 다른 리전에 IaC 기반 인프라를 재구성하여 즉각 대응할 수 있는 구조를 설계합니다.

    • 3rd Party 솔루션 및 Private 클라우드를 사용하는 경우, 이를 준수하는 대응 방안을 제시합니다.

    이러한 방향으로 ML/DL 분석 환경을 개발하고, MLOps 체계를 구현함으로써, KB는 AI 기반의 분석 능력과 모델 운영 효율성을 향상시키며, 리스크 관리 및 신규 서비스 개발을 가속화할 수 있습니다.

  • 4. Vector Store (벡터 저장소)

    4.1. 기술 설명

Vector Store는 벡터 데이터를 저장하고, 유사도 검색을 위한 고성능 저장소입니다. 이는 텍스트, 이미지, 음성 등 다양한 형태의 데이터를 벡터로 변환하여 저장하며, 유사도 기반의 빠른 검색을 가능하게 합니다. Vector Store는 AI Agent가 대규모 데이터를 효율적으로 검색하고, 추천 시스템 및 질문-답변(Q&A) 기능을 구현하는 데 핵심적인 역할을 합니다.

첨부된 문서에 따르면, Vector Store는 AI 서빙 환경에서 저지연(low latency)의 데이터 접근을 지원하며, 인덱스 생성을 통해 빠른 검색이 가능합니다. 또한, Vector Search는 AI 서빙을 위한 최소 운영 기준으로, 마스터 노드와 데이터 노드가 멀티 AZ 분산 방식으로 구성되어 고가용성과 안정성을 확보합니다. 이러한 구조는 비정형 데이터의 빠른 접근 및 분석을 가능하게 하며, AI 친화적인 데이터 서빙 환경을 구축하는 데 기여합니다.

Vector Store는 자연어, 이미지, 비정형 데이터를 벡터로 변환해 저장하고, 일관성 있게 AI 학습 및 추론에 활용할 수 있도록 Feature Store와 연계되며, GraphRAG와 같은 고급 검색 기술을 활용하여 데이터의 의미를 이해하고 더 정확한 응답을 제공할 수 있습니다.

이와 같은 기술적 특성으로 인해 Vector Store는 KB의 AI Agent 및 생성형 AI 기반의 차세대 업무 지원 체계 구축에 중요한 기반 기술로 활용되고 있습니다.

      • 이러한 벡터 데이터베이스는 AI Agent나 ML/DL 모델에서 대규모 비정형 데이터를 처리하는 데 필수적인 기반 기술이다.

      • Vector Search는 의료 문서 처리, 고객 지원, 챗봇, 추천 시스템 등 다양한 분야에서 활용될 수 있으며, KB의 리스크 Agent에서도 의료 지식 시스템이나 고객 데이터 분석에 적용될 수 있다.

      • Vector Search 기능은 LLM-Ready 데이터를 기반으로 의미 기반 검색, 코호트 분석, 의사결정 지원 등을 가능하게 한다.

      • Vector Store는 ML/DL 모델의 입력 및 출력 데이터를 저장하고, AI Agent가 이를 기반으로 대규모 데이터 검색 및 추천을 수행할 수 있도록 지원한다.

      • Vector Store는 시맨틱 레이어와 결합하여 AI가 이해하고 활용할 수 있는 구조를 제공하며, KB의 시맨틱 데이터와도 연계될 수 있다.

      • Vector Store는 SQL 기반의 데이터 처리와 NLP, 벡터 검색을 결합하여 의미 기반의 데이터 분석을 가능하게 한다.

    • AI Agent의 대규모 데이터 검색 및 추천 지원

      • AI Agent는 KB의 리스크 Agent와 CDSS (Clinical Decision Support System)에서 활용될 수 있으며, 의료 지식 시스템이나 고객 지원 시스템에서도 활용될 수 있다.

      • AI Agent는 Vector Store에 저장된 의료 문서, 고객 데이터, 금융 데이터 등을 기반으로 의미 기반의 검색, 코호트 분석, 의사결정 지원 등을 제공한다.

5. MLOps

5.1. 기술 설명 (MLOps)

MLOps는 머신러닝(ML) 및 딥러닝(DL) 모델의 전체 생명주기(개발 → 테스트 → 배포 → 모니터링 → 재학습)를 자동화하고 표준화하는 운영 체계입니다. 이는 데이터 과학자, 개발자, DevOps 팀 간의 협업을 체계적으로 지원하여, 모델 개발 효율성과 운영 안정성을 높이도록 설계되었습니다.

첨부된 자료에 따르면, MLOps는 다음과 같은 핵심 기능을 수행합니다:

  1. 자동화된 ML 파이프라인

    • 데이터 추출, 전처리, 학습, 평가, 배포, 모니터링, 재학습 등 ML 모델의 전 과정을 자동화.

    • 분석가가 작성한 코드 이력 관리를 위한 repository 구축.

    • Feature 공유 및 확산, 모델 품질 평가, 재학습 자동화, 예측 API 서비스 지원.

  2. 모델 운영 자동화

    • ML 분석 인프라 및 솔루션 도입을 통해 모델 운영이 체계적으로 관리됨.

    • ML 파이프라인 운영 관리와 자원 생성/회수 기능을 지원하여, 효율적인 자원 활용이 가능.

    • 현행 ML 모델 파이프라인을 리스크 Agent 내로 이관하여 통합 관리.

  3. 표준화 및 거버넌스

    • MLOps 체계 구축을 통해 모델의 품질, 성능, 재학습 주기 등을 표준화.

    • 모델 품질 평가, 재학습, 예측 API 서비스 지원 등이 자동화되어 운영 효율성 향상.

    • DataOps 및 MLOps 체계를 기반으로 데이터 수집, 처리, 모델 운영이 체계적으로 이뤄짐.

  4. 리스크 Agent 연계

    • 리스크 Agent에 필요한 ML/DL 모델 개발 및 운영 환경을 제공.

    • 리스크 관련 모델(예: 기업여신, 신용평가 등)이 MLOps 기반으로 API 배포 및 관리됨.

    • 데이터 파이프라인 마이그레이션을 통해 검증된 모델이 확산되어 실시간 리스크 관리가 가능.

  5. 보안 및 데이터 거버넌스

    • 개인정보 및 민감 데이터 보호를 위해 마스킹, 해시 등 데이터 보호 체계 구축.

    • 데이터 접근 로깅, 규제 준수 모니터링, 데이터 품질 정의 및 평가 체계 강화.

    • Data as a Product(DaaP) 모델로 데이터 제품화를 통해 품질 기반의 데이터 공유 및 활용 강화.

  6. 업무 효율성 및 경쟁력 확보

    • Agentic AI 및 생성형 AI 기술을 활용한 차세대 업무 지원 체계 구축.

    • 디지털 트윈, 디지털 워크플로우, 인사이트 기반 신규 서비스 창출 등으로 AI 기반 비즈니스 혁신 가속화.

요약하면, MLOps는 단순히 ML 모델을 개발하고 배포하는 것을 넘어, 모델의 전 생명주기 운영을 체계화하고 자동화하여, AI 기반의 비즈니스 혁신을 뒷받침하는 핵심 기술입니다. 이는 KB의 Modern Data Platform 및 리스크 Agent 구축과도 밀접하게 연계되어, 데이터 기반의 경쟁력 확보와 운영 효율성 향상에 기여합니다.

5.2. RFP 기반 구현 방향

  • ML/DL 모델의 전주기 운영을 지원하는 MLOps 체계를 구축합니다. 이는 데이터 추출, 분석, 준비, 학습, 평가, 검증, 서빙, 모니터링, 재학습 등 ML 모델의 전체 운영 과정을 자동화하고 효율적으로 관리하는 것을 목표로 합니다.

  • ML/DL 분석 인프라 및 솔루션을 도입하여, 분석가가 작성한 코드 이력 관리를 위한 리포지토리를 구축하고, Feature 확산 및 공유, 모델 품질 평가, 재학습, 예측 API 서비스 지원 등을 체계적으로 수행합니다.

  • KubeFlow, MLFlow, Airflow 등 오픈소스 기반의 MLOps 도구를 활용하여, ML 파이프라인의 자동화, 모델 버전 관리, 실험 추적, 재현성 확보 등 핵심 MLOps 기능을 구현합니다.

  • 리스크 Agent 내 ML 모델 파이프라인 개발 및 운영을 지원합니다. ML/DL 기반의 리스크 모델(예: 기업여신, 신용평가 등)을 구축하고, 이를 기반으로 데이터, API 배포 등 파이프라인 이관을 수행하여 실제 업무에 적용할 수 있도록 합니다.

  • ML 분석 자원 생성 및 회수 기능을 구현하여, 자원의 효율적 배분과 재사용을 지원하며, 현행 분석가/프로젝트 모델 파이프라인의 이관을 원활히 수행합니다.

  • 자동화된 ML 파이프라인 운영 관리를 통해 ML 모델의 품질과 성능을 지속적으로 모니터링하고, 필요 시 재학습 및 업데이트를 수행하여 높은 수준의 정확도와 신뢰성을 확보합니다.

  • 리스크 Agent 내 ML 모델 개발 지원을 위해, ML/DL 분석 솔루션과 MLOps 체계를 결합한 개발 환경을 제공하여, 높은 수준의 모델 성능과 빠른 개발 주기를 달성합니다.

이와 같은 MLOps 구현 방향은 KB의 리스크 Agent 중심의 데이터 현대화 전략에 부합하며, 데이터 기반의 AI 경쟁력 확보와 AI 기반 업무 지원 체계 구축을 가속화하는 데 기여할 것입니다.

5.3. MLOps 구현을 위한 핵심 기술 요소

MLOps는 머신러닝(Machine Learning, ML) 및 딥러닝(Deep Learning, DL) 모델의 전 생명주기를 체계적으로 관리하는 DevOps 기반의 접근 방식으로, 데이터 전처리, 모델 개발, 테스트, 배포, 모니터링, 재학습 등 모든 과정을 자동화하고 표준화하여 AI 모델의 신뢰성과 활용성을 높이는 데 목적이 있습니다. 아래는 MLOps 구현을 위한 핵심 기술 요소를 첨부된 자료를 바탕으로 정리한 내용입니다.

1. ML/DL 모델 전 생명주기 관리

MLOps의 핵심은 ML/DL 모델의 전 생명주기를 체계적으로 관리하는 것입니다. 이는 다음과 같은 주요 단계를 포함합니다:

  • 데이터 추출 및 전처리: 브론즈 레이어(원천 데이터)에서 실버 레이어(정제된 데이터)로의 전환.

  • 모델 개발 및 학습: 다양한 언어와 도구를 활용한 모델 학습.

  • 모델 평가 및 검증: 품질 평가, 재학습 기준 설정.

  • 모델 배포: 예측 API 서비스, 서빙 환경 구축.

  • 모델 모니터링: 성능 감시, 데이터 편향 감지, 재학습 필요성 판단.

  • 모델 회수 및 업데이트: 자원 회수, 새로운 데이터 반영을 통한 모델 업데이트.

이러한 과정은 ML/DL 분석 인프라 및 솔루션 도입과 자동화된 ML 파이프라인 운영 관리를 통해 실현됩니다. 특히, ML 분석 자원 생성 및 회수 기능은 효율적인 자원 관리에 기여합니다.

2. MLOps 도구 활용

MLOps의 핵심 기술 요소 중 하나는 다양한 MLOps 도구를 활용하여 모델 개발 및 관리의 자동화와 표준화를 실현하는 것입니다. 주요 도구는 다음과 같습니다:

  • KubeFlow: Kubernetes 기반의 ML 워크플로우 관리 도구로, ML 모델의 개발, 테스트, 배포, 모니터링을 한 곳에서 관리할 수 있습니다.

  • MLFlow: ML 모델의 실험, 패키징, 배포, 추적을 위한 오픈소스 플랫폼입니다. 모델의 버전 관리, 실험 추적, 모델 레지스트리 기능을 제공합니다.

  • Airflow: 데이터 파이프라인 및 배치 처리를 자동화하는 도구로, MLOps에서 모델 학습, 평가, 재학습 작업을 스케줄링하고 관리하는 데 사용됩니다.

이러한 도구들은 ML 분석 솔루션 도입 및 자동화된 ML 파이프라인 운영 관리에 활용되며, 특히 리스크 Agent 내 ML 모델 파이프라인 개발 지원에 중요한 역할을 합니다.

3. ML/DL 모델 배포 및 모니터링

ML/DL 모델은 개발 완료 후, 실제 업무 환경에 배포되어야 의미 있는 가치를 창출할 수 있습니다. MLOps에서는 다음과 같은 배포 및 모니터링 기술 요소가 중요합니다:

  • 모델 서빙: 예측 API 서비스를 통해 모델을 실시간 또는 배치 방식으로 제공.

  • 모델 모니터링: 성능 감시, 데이터 편향 감지, 재학습 필요성 판단.

  • 재학습 자동화: 데이터 변화에 따라 자동으로 모델을 재학습하는 프로세스.

이러한 기능은 **ML 모델 운영 자동화(MLOps)**의 핵심 요소로, 리스크 Agent와의 연동을 통해 모델의 신뢰성과 효과성을 높일 수 있습니다.

4. 리스크 Agent와의 연동

리스크 Agent는 ML/DL 모델의 효과성을 검증하고, 실제 업무 환경에서의 성능을 모니터링하는 데 중요한 역할을 합니다. MLOps에서는 다음과 같은 방식으로 리스크 Agent와 연동됩니다:

  • 모델 이관: ML 분석가가 개발한 모델을 리스크 Agent 내에서 실행하고 모니터링.

  • 데이터 검증: Agent를 통해 모델이 사용하는 데이터의 품질과 적절성을 검증.

  • 리스크 기반 모델 개선: Agent가 수집한 리스크 데이터를 바탕으로 모델을 개선하고 재학습.

이러한 연동은 ML 분석 인프라 및 솔루션 도입을 통해 구현되며, 리스크 Agent 내 ML 모델 파이프라인 개발 지원에 기여합니다.

5. 데이터 파이프라인 자동화

ML/DL 모델의 성능은 입력 데이터의 품질에 크게 의존합니다. 따라서 MLOps에서는 다음과 같은 데이터 파이프라인 자동화 기술 요소가 중요합니다:

  • 데이터 추출 및 가공: Spark, DuckDB, Trino 등의 도구를 활용한 데이터 처리.

  • 데이터 버전 관리: 데이터의 버전을 관리하고, Time Travel 기능을 통해 데이터 복구.

  • 데이터 품질 검증: 데이터 오류 감지 및 품질 평가.

이러한 기능은 ML/DL 분석 인프라 및 솔루션 도입을 통해 구현되며, ML 분석 인프라 및 솔루션 도입은 ML/DL 분석 인프라 및 솔루션 도입을 통해 구현됩니다.

결론

MLOps는 ML/DL 모델의 전 생명주기를 체계적으로 관리하는 데 중요한 역할을 합니다. 핵심 기술 요소는 ML/DL 모델 전 생명주기 관리, MLOps 도구 활용, ML/DL 모델 배포 및 모니터링, 리스크 Agent와의 연동, 데이터 파이프라인 자동화 등입니다. 이러한 요소들을 적절히 조합하고 구현함으로써, ML/DL 모델의 신뢰성과 활용성을 높이며, 데이터 기반의 AI 경쟁력을 확보할 수 있습니다.

6. 데이터 카탈로그 강화

6.1. 기술 설명

데이터 카탈로그는 데이터 자산을 탐색, 이해, 활용할 수 있도록 메타데이터를 관리하는 시스템입니다. RFP에서는 정형과 비정형 데이터를 포괄하는 데이터 카탈로그를 구축하고, 비정형 데이터의 메타 관리 표준 체계를 도입하는 것을 강조하고 있습니다.

6.2. RFP 기반 구현 방향

  • DataHub (오픈소스) 기반으로 커스텀한 데이터 카탈로그를 운영 중이며, 이를 고도화하여 행내/외 데이터를 포괄적으로 관리합니다.

  • 비정형 데이터의 메타 관리 표준 체계를 도입하여 데이터의 탐색 및 활용을 용이하게 합니다.

6.3. 핵심 기술 요소

  • DataHub 기반 데이터 카탈로그 고도화

  • 정형/비정형 데이터 포괄 관리

  • 비정형 데이터 메타 관리 표준 체계 도입

7. 시맨틱 데이터 저장소

7

1. 기술 설명

시맨틱 데이터 저장소(Semantic Data Repository)는 데이터에 의미(Semantics)와 관계(Relationships)를 부여하여 AI Agent가 데이터를 이해하고 활용할 수 있도록 구조화된 데이터를 저장하고 관리하는 저장소입니다. 이 저장소는 단순히 데이터를 저장하는 것을 넘어, 데이터 간의 의미적 맥락, 연결성, 도메인별 특징 등을 체계적으로 정의하고 관리함으로써 AI가 데이터를 자연스럽게 해석하고 추론하는 데 기반을 제공합니다.

이러한 저장소는 특히 정형 데이터(Relational Data)와 비정형 데이터(Unstructured Data)의 통합 활용에 유용하며, 지식그래프(Knowledge Graph), 벡터 DB(Vector DB), 시맨틱 레이어(Semantic Layer) 등의 기술과 결합되어 AI Agent의 성능 향상을 도모합니다.

주요 기술적 특징

  1. 의미 기반 데이터 모델링

    • 데이터는 단순한 값을 넘어서, 도메인별 의미(예: 고객, 거래, 리스크), 관계(예: 고객-거래, 거래-리스크)를 포함하여 저장됩니다.

    • 예를 들어, 고객 정보는 단순히 이름, ID, 나이 등이 아니라, \"고객-제품-거래\"와 같은 의미적 연결성을 포함하여 저장됩니다.

  2. AI 친화적 데이터 구조

    • AI Agent가 데이터를 학습 및 추론에 활용하기 쉽도록, 벡터화(Vectorization), 특징 추출(Feature Extraction), 메타데이터(Metadata) 등을 포함하여 데이터를 구조화합니다.

    • Vector DB(예: Milvus, Pinecone)와 같은 저장소를 활용하여 이미지, 텍스트, 음성 등 비정형 데이터도 의미 기반으로 저장 및 검색이 가능합니다.

  3. 시맨틱 레이어 구성

    • 데이터를 명확한 의미 기반으로 메타 정보(Metadata)와 관계(Relationships)를 재구축하여 AI가 쉽게 이해할 수 있도록 구성합니다.

    • 예: 고객 데이터는 \"고객-금융상품-거래-리스크\"와 같은 의미적 맥락에 따라 구조화되어 저장되며, SQL 기반의 단일 데이터 소스가 아닌 데이터 패브릭(Data Fabric) 기반으로 확장되어 활용됩니다.

  4. 지식그래프 기반의 의미 연결

    • 지식그래프(Knowledge Graph)를 활용하여 데이터 간의 의미적 연결성을 시각화하고 관리합니다.

    • 예: RDB와의 차별점이 명확한 지식그래프 기반의 데이터 모델을 통해, GraphRAG(Graph-based Retrieval Augmented Generation)와 같은 기술을 활용한 보다 정확한 AI 응답이 가능해집니다.

  5. 데이터 통합 및 활용

    • 정형 데이터(RDB, SQL)와 비정형 데이터(문서, 영상, 음성)를 통합하여 AI가 활용할 수 있는 AI-Ready 데이터를 제공합니다.

    • 데이터 카탈로그(Data Catalog)와 메타데이터 관리 시스템을 통해 데이터의 의미, 출처, 사용 목적 등을 체계적으로 관리합니다.

기대 효과

  • AI Agent 성능 향상: 의미 기반 데이터를 활용하여 AI가 더 정확하고 자연스럽게 데이터를 해석하고 추론할 수 있습니다.

  • 데이터 품질 강화: 의미와 관계를 정의함으로써 데이터의 일관성, 정확성, 완전성이 향상됩니다.

  • AI 친화적 데이터 제공: Vector DB, Feature Store, Graph DB 등 다양한 저장소를 활용하여 AI가 데이터를 학습 및 추론에 쉽게 활용할 수 있습니다.

  • 데이터 연결성 강화: 시맨틱 데이터 저장소를 통해 다양한 데이터 소스가 의미 기반으로 연결되어, 데이터 패브릭(Data Fabric) 기반의 통합 데이터 활용이 가능합니다.

  • 고객 인사이트 도출: 의미 기반 데이터를 활용하여 고객 행동, 리스크, 트렌드 등을 분석하여 새로운 인사이트와 서비스를 도출할 수 있습니다.

관련 구현 사례

  • KB의 시맨틱 데이터 저장소 구축
    KB는 시맨틱 레이어, 지식그래프, Vector DB 등을 활용하여 AI Agent가 데이터를 이해하고 활용할 수 있는 저장소를 구축하고 있습니다.

    • GraphRAG 기반의 AI 응답 방식을 도입하여, 환각(Hallucination)을 줄이고 더 정확한 응답을 제공합니다.

    • 리스크 Agent 내에서 활용되는 ML 모델 파이프라인을 지원하며, Vector DB를 활용한 비정형 데이터 서빙을 강화합니다.

  • Vector DB 및 Feature Store 활용

    • Vector DB(Vector Database)는 이미지, 텍스트, 음성 등 비정형 데이터를 벡터 형태로 저장하고, 유사도 검색(Similarity Search)을 통해 AI가 데이터를 쉽게 활용할 수 있도록 지원합니다.

    • Feature Store는 AI 학습 및 추론에 필요한 데이터를 일관성 있게 제공함으로써, 반복적인 데이터 가공 과정을 줄이고 AI 성능을 향상시킵니다.

결론

시맨틱 데이터 저장소는 단순히 데이터를 저장하는 저장소를 넘어, AI Agent가 데이터의 의미를 이해하고 활용할 수 있도록 구조화된 데이터를 저장하고 관리하는 핵심 인프라입니다.
이 저장소는 의미 기반 데이터 모델링, AI 친화적 데이터 구조, 시맨틱 레이어, 지식그래프, Vector DB 등의 기술을 활용하여, AI가 데이터를 자연스럽게 해석하고 추론하는 데 기반을 제공합니다.
KB는 이러한 저장소를 구축하여 AI Agent 성능 향상, 데이터 품질 강화, 고객 인사이트 도출 등을 목표로 하고 있으며, Agent 중심의 데이터 현대화를 통해 AI 기반의 디지털 트윈, 디지털 워크플로우, 차세대 업무지원 체계 등을 구현하고 있습니다.

8. Vector Search (벡터 검색)

8.1. 기술 설명

Vector Search는 **벡터 데이터베이스(Vector DB)**를 기반으로 유사도 기반의 검색 기술을 구현하는 방식입니다. 이는 데이터를 수치화된 벡터로 표현하여, 유사한 의미를 가진 데이터를 빠르게 탐색하고 추천하는 데 활용됩니다. 특히 대규모 텍스트, 이미지, 음성 데이터 등 멀티모달 데이터 처리에 적합하며, AI가 이러한 데이터를 이해하고 활용하는 데 핵심적인 역할을 합니다.

주요 기능 및 특징

  1. 벡터 기반 유사도 검색

    • 데이터를 벡터로 변환하여, 유사도 계산을 통해 빠르고 정확한 검색이 가능합니다.

    • 예: 사용자의 질의를 벡터로 변환하고, 이에 가장 유사한 문서나 이미지를 빠르게 탐색.

  2. AI 활용 최적화

    • Vector DB는 AI가 학습 및 추론에 필요한 특징 벡터(Feature Vector)를 저장 및 관리하는 데 활용됩니다.

    • Feature Store와 연계하여 반복적으로 사용되는 학습 및 추론 입력 데이터를 일관성 있게 제공할 수 있습니다.

  3. 저지연(Low Latency) 처리

    • 인메모리 기반의 데이터 서빙(예: Redis)과 결합하여, 실시간 응답이 필요한 시스템에서 효과적으로 활용됩니다.

    • 예: 고객 행동 로그를 실시간으로 수집하여 분석하고, 실시간 추천을 제공.

  4. 비정형 데이터 처리

    • 이미지, 텍스트, 음성 등 다양한 유형의 비정형 데이터를 벡터로 변환하여 일관된 검색 및 분석이 가능합니다.

    • GraphRAG(Graph Retrieval-Augmented Generation) 등과 결합하면, 의미와 관계를 고려한 검색이 가능해집니다.

  5. 고가용성 및 확장성

    • PB(페타바이트) 단위의 대규모 데이터 저장 및 분산 처리가 가능하며, SLA 기반의 성능과 보안을 지원합니다.

    • 시스템이 중단 없이 확장되어, 사용량 증가에 유연하게 대응할 수 있습니다.

활용 사례

  • AI 추천 시스템: 사용자에게 맞춤형 콘텐츠, 상품, 서비스 추천.

  • 지식 그래프 검색: 의미 기반으로 연결된 데이터를 빠르게 탐색.

  • 이미지/텍스트 검색: 이미지나 텍스트를 입력으로 하여 유사한 결과 검색.

  • 리스크 모델링: 리스크 에이전트를 활용한 데이터 분석 및 모델링.

관련 기술 및 도구

  • Vector DB: FAISS, Milvus, Pinecone 등.

  • Feature Store: Feast, Tecton 등.

  • ML/DL 분석 도구: Spark, Trino, DuckDB 등.

  • 인메모리 서빙: Redis, Memcached 등.

  • MLOps: ML 파이프라인 자동화, 모델 품질 평가, 재학습 등.

구현 전략

  • 데이터 변환: 자연어, 이미지, 음성 등을 벡터로 변환 (예: 임베딩 모델 활용).

  • 벡터 저장소 설계: 고가용성, 확장성, 보안 기능을 갖춘 Vector DB 구축.

  • 검색 및 추천 엔진: 유사도 계산 알고리즘(코사인 유사도, 유클리드 거리 등) 기반으로 실시간 검색 및 추천 기능 구현.

  • 모니터링 및 최적화: 자원 사용량, 검색 성능, 사용자 피드백 등을 모니터링하여 지속적인 개선.

Vector Search는 AI 친화적인 데이터 플랫폼 구축에 핵심적인 기술로, 데이터 기반의 AI 경쟁력 확보와 다양한 AI Biz 적용을 가속화할 수 있습니다.

    • Multi-Modal Embedding Fusion – 텍스트, 이미지, 음성 등 다양한 데이터 유형을 통합적으로 임베딩하여, 복합적인 의미를 반영하는 벡터를 생성합니다.

    • GPU 가속 기반의 인덱싱(Indexing) – ANN (Approximate Nearest Neighbor) 알고리즘을 기반으로, 유사도 검색 시 수백만 개의 벡터를 초단위로 처리 가능.

    • Query Expansion & Semantic Boosting – 사용자 쿼리를 이해하여 의미적으로 유사한 키워드나 문장을 자동으로 확장하고, 관련성 높은 결과를 우선적으로 제공.

    • Context-Aware Filtering – 사용자의 이전 검색 이력, 역할, 목적에 따라 검색 결과를 커스터마이징하여 개인화된 추천을 제공.

8.3. 핵심 기술 요소

벡터 검색은 AI 기반 시스템에서 비정형 데이터(텍스트, 이미지, 음성 등)를 효과적으로 처리하고 의미를 기반으로 정보를 검색하는 데 핵심적인 역할을 합니다. 이를 구현하기 위해 다음과 같은 핵심 기술 요소가 필요합니다. 특히, 네오알리의 차별화된 지식 파서(Knowledge Parser)와 RAG(Retrieval-Augmented Generation) 기술은 이 과정에서 중요한 역할을 합니다.

1. Vector Database 도입 (예: FAISS, Milvus, Weaviate 등)

벡터 데이터베이스는 고차원 벡터를 저장하고 효율적으로 검색할 수 있는 시스템입니다. 이는 AI가 의미 기반으로 데이터를 탐색할 수 있도록 합니다. 네오알리의 지식 파서는 데이터를 의미 단위로 분해하고 이를 벡터로 변환하여 Vector DB에 저장합니다. 이 과정에서 자연어 처리(NLP) 기술을 활용해 텍스트의 의미를 정확히 포착하고, 의미적으로 유사한 데이터를 클러스터링하여 저장합니다.

네오알리의 차별점: 일반적인 Vector DB에 비해, 네오알리는 의미 기반 벡터 인코딩을 통해 단순한 텍스트 유사도가 아닌 실질적인 의미의 유사도를 반영합니다. 예를 들어, \"고객 만족도\"라는 키워드는 단순히 텍스트 유사도가 아닌, 고객의 감정, 행동, 이력 등을 종합적으로 반영한 의미 벡터로 인코딩됩니다.

2. Vector Search 기능 구현

Vector Search는 사용자의 질의를 벡터로 변환한 후, 이 벡터와 가장 유사한 벡터를 Vector DB에서 빠르게 검색하는 기술입니다. 이 과정에서 Cosine 유사도, ANN(Approximate Nearest Neighbor) 알고리즘 등이 사용됩니다. 네오알리의 RAG 기술은 이 과정에서 검색된 벡터의 의미적 관련성을 고려하여 질의에 대한 정확한 응답을 생성합니다.

네오알리의 RAG 기술: RAG는 Retrieval과 Generation을 결합한 기술로, 사용자의 질의에 대해 Vector DB에서 가장 관련성 높은 정보를 검색한 후, 이를 기반으로 생성형 AI가 자연어로 응답을 생성합니다. 이는 단순한 키워드 검색을 넘어, 의미와 맥락을 고려한 정확한 정보 제공이 가능하게 합니다.

3. Vector Store와 ML/DL, 레이크하우스의 통합

Vector Store는 Vector DB에 저장된 벡터 데이터를 ML/DL 모델과 연결해주는 역할을 합니다. 네오알리의 지식 파서는 레이크하우스에 저장된 정형/비정형 데이터를 의미 기반으로 파싱하고, 이를 Vector Store에 저장합니다. 이후 ML/DL 모델은 Vector Store에서 벡터를 로드해 학습 및 추론에 활용할 수 있습니다.

네오알리의 차별점: 일반적인 Vector Store는 단순히 데이터를 저장하는 데 그치지만, 네오알리는 데이터의 의미 구조를 유지하면서도 Vector DB와 ML/DL 모델 간의 통합이 가능합니다. 예를 들어, 고객 데이터는 이름, 이력, 행동 등 의미 단위로 파싱되어 Vector DB에 저장되고, 이를 기반으로 고객 분류, 추천, 감정 분석 등의 AI 모델이 학습됩니다.

4. AI Agent의 대규모 데이터 검색 및 추천 지원

AI Agent는 사용자의 입력을 기반으로 Vector DB에서 가장 관련성 높은 정보를 검색하고, 이를 기반으로 추천이나 응답을 생성합니다. 네오알리의 지식 파서와 RAG 기술은 AI Agent가 단순한 키워드 매칭이 아닌 의미와 맥락을 기반으로 정확한 정보를 추출 및 생성할 수 있도록 합니다.

네오알리의 차별점: 일반적인 AI Agent는 키워드 매칭이나 단순한 유사도 검색에 의존하지만, 네오알리는 의미 기반 Vector Search와 RAG 기술을 통해 사용자의 질의에 대한 깊은 이해와 정확한 응답 생성이 가능합니다. 예를 들어, 고객이 \"제품 추천을 원합니다\"라고 입력하면, 네오알리의 시스템은 고객의 이력, 관심사, 행동 등을 종합적으로 분석해 가장 적합한 제품을 추천합니다.

5. 시맨틱 레이어와 지식 그래프 연계

네오알리의 지식 파서는 데이터를 의미 기반으로 구조화하여 시맨틱 레이어를 구축합니다. 이 시맨틱 레이어는 데이터 간의 관계를 명확히 정의하고, 이를 기반으로 지식 그래프를 생성합니다. 이 지식 그래프는 Vector Search와 RAG 기술과 연계되어, AI가 데이터의 의미적 맥락을 이해하고, 더 정확한 정보를 제공할 수 있도록 합니다.

네오알리의 차별점: 일반적인 시맨틱 레이어는 데이터 간의 단순한 연결이지만, 네오알리는 의미 기반 벡터 인코딩과 지식 그래프를 결합해, 데이터의 의미와 관계를 정확히 포착합니다. 이는 AI가 단순한 정보 제공이 아닌, 사용자의 질문에 대한 깊은 이해와 맞춤형 응답이 가능하게 합니다.

6. GraphRAG 기반의 환각 해소

GraphRAG는 RAG 기술에 지식 그래프를 추가한 형태로, AI가 단순히 텍스트를 기반으로 응답하는 것이 아니라, 데이터 간의 관계와 맥락을 고려한 더 정확한 응답을 생성할 수 있도록 합니다. 네오알리의 시스템은 GraphRAG 기술을 통해 AI가 데이터의 의미적 구조를 이해하고, 사용자의 질문에 대해 더 정확하고 일관된 응답을 제공합니다.

네오알리의 차별점: GraphRAG는 AI가 단순히 텍스트를 기반으로 응답하는 것이 아니라, 데이터 간의 관계와 맥락을 고려한 더 정확한 응답을 생성할 수 있도록 합니다. 이는 AI가 단순한 정보 제공이 아닌, 사용자의 질문에 대한 깊은 이해와 맞춤형 응답이 가능하게 합니다.

7. Vector DB와 AI Agent의 결합

Vector DB는 단순히 데이터를 저장하는 것이 아니라, AI Agent가 데이터를 의미 기반으로 검색하고, 이를 기반으로 추천, 분석, 응답을 생성할 수 있도록 합니다. 네오알리의 지식 파서와 RAG 기술은 Vector DB와 AI Agent를 결합해, 사용자의 질의에 대해 의미와 맥락을 고려한 정확한 응답을 생성합니다.

네오알리의 차별점: 일반적인 Vector DB는 AI Agent와의 결합이 어렵지만, 네오알리는 의미 기반 Vector Search와 RAG 기술을 통해 AI Agent가 Vector DB에서 데이터를 의미 기반으로 검색하고, 이를 기반으로 정확한 응답을 생성할 수 있도록 합니다.

결론

네오알리의 지식 파서와 RAG 기술은 Vector Search 기반의 AI 시스템에서 핵심적인 역할을 합니다. 이 기술은 단순한 텍스트 유사도 검색을 넘어, 의미와 맥락을 고려한 정확한 정보 제공이 가능하게 합니다. 이는 AI Agent가 사용자의 질문에 대해 깊은 이해와 맞춤형 응답을 제공할 수 있도록 합니다.

DataOps는 데이터 파이프라인의 전 과정을 효율적으로 관리하고 최적화하기 위한 운영 체계로, 데이터 수집, 전처리, 통합, 분석, 배포 및 모니터링에 걸쳐 자동화와 표준화를 적용합니다. 이는 데이터의 신뢰성, 일관성, 품질을 보장하며, 데이터 기반 의사결정의 속도와 정확도를 향상시킵니다. 첨부 문서에 따르면 DataOps는 다음과 같은 핵심 요소를 포함합니다:

  • 데이터 표준 수집 체계 개발: 실시간, 배치, API를 통해 표준화된 데이터 수집 체계를 구축하며, 행내 비정형 데이터를 활용하기 위해 WiseNet, KB-Drive API 등을 연계합니다. 레가시 데이터 수집 시 호환성을 유지하는 것도 중요합니다.

  • DataOps 체계 구축: 메타데이터 기반의 데이터 파이프라인 운영 관리, 내부 ↔ 클라우드 간 최적화된 데이터 전송 구현, 다양한 데이터 유형(정형, 비정형, 실시간)을 저장 및 처리할 수 있는 레이크하우스 아키텍처를 도입합니다.

  • 오픈 테이블 포맷 적용: Iceberg, DeltaLake 등과 같은 오픈 테이블 포맷을 활용하여 멀티엔진으로 전환이 가능한 구조를 설계하여 호환성을 높이고, 데이터 이동 및 유연한 확장을 지원합니다.

  • CI/CD 환경 구축: 소스 코드, 파이프라인, 인프라 구성 코드 등을 포함한 CI/CD 환경을 구축하여 데이터 처리 및 인프라 변경을 자동화합니다.

  • Data Governance 강화: 데이터의 신뢰성, 보안, 활용성을 보장하기 위해 데이터 거버넌스를 수립하며, 데이터 카탈로그의 적용 범위를 비정형 데이터까지 확장합니다.

MLOps는 ML/DL 모델의 전 생명주기(데이터 추출, 분석, 학습, 평가, 검증, 서빙, 모니터링)를 자동화 및 표준화하는 운영 체계입니다. 이는 모델의 품질, 재현성, 안정성, 규제 준수 등을 보장하며, ML 모델의 빠른 개발 및 배포를 지원합니다. 첨부 문서에 따르면 MLOps는 다음과 같은 주요 기능을 포함합니다:

  • ML/DL 분석 인프라 및 솔루션 도입: KubeFlow, 레이크하우스 분석 솔루션, CSP 서비스 등을 활용하여 ML/DL 분석 인프라를 구축합니다. 이는 데이터 추출, 준비, 학습, 평가, 검증, 서빙, 모니터링 등을 자동화합니다.

  • ML 모델 품질 평가 및 재학습: ML 모델의 품질을 평가하고, 재학습을 통해 모델의 성능을 지속적으로 개선합니다.

  • Feature 확산 및 공유: ML 모델에서 사용된 Feature를 확산 및 공유하여 다른 프로젝트나 부서에서도 활용할 수 있도록 합니다.

  • 예측 API 서비스 지원: ML 모델을 기반으로 예측 API 서비스를 제공하여 실시간으로 분석 결과를 반환합니다.

  • ML 파이프라인 운영 관리: ML 분석 인프라 및 파이프라인을 자동화하여 운영 관리합니다. 현행 분석가/프로젝트 모델 파이프라인을 이관 지원하며, 리스크 Agent 내 ML 모델 파이프라인 개발도 지원합니다.

  • ML 분석 자원 생성 및 회수: ML 분석에 필요한 자원을 생성하고, 사용 후 자원을 회수하여 효율적으로 관리합니다.

DataOps와 MLOps는 서로 밀접하게 연계되어 있으며, DataOps는 ML/DL 모델의 학습 및 추론에 필요한 데이터를 안정적이고 신뢰성 있게 제공하고, MLOps는 해당 데이터를 기반으로 ML 모델을 개발, 배포, 운영합니다. 이를 통해 AI 기반의 업무 지원 체계가 구축되며, 디지털 트윈, 디지털 워크플로우 등 미래형 업무 방식의 기반을 마련할 수 있습니다.

2. RFP 기반 구현 방향

2.1 DataOps 체계 구축

  • 표준화된 데이터 수집 체계 구축:

    • 실시간, 배치, API를 통한 표준화된 데이터 수집 프로세스를 수립합니다.

    • KB WiseNet, KB-Drive와 연계하여 행내 비정형 데이터를 자동으로 수집하고, 레가시 시스템과의 호환성을 유지합니다.

    • 외부 공공기관 및 리스크 Agent가 필요한 데이터는 검증된 API를 통해 자동 수집 및 적재합니다.

  • 데이터 통합 및 저장:

    • 클라우드 기반의 레이크하우스 아키텍처를 도입하여 정형, 비정형, 실시간 데이터를 통합 저장합니다.

    • Iceberg, DeltaLake 등 오픈 테이블 포맷을 활용해 멀티엔진 처리 및 데이터 이동 최소화를 기반으로 운영합니다.

    • Bronze, Silver, Gold 레이어를 구분하여 데이터의 원본 보존, 가공, 최종 활용 단계를 체계화합니다.

  • DataOps 자동화 및 모니터링:

    • 데이터 파이프라인의 자동화를 통해 데이터 품질 검증, 메타데이터 추출, 데이터 카탈로그 업데이트 등을 실시간으로 처리합니다.

    • 데이터 흐름, 로깅, 오류 감지 및 알림 시스템을 구축하여 데이터 품질과 처리 투명성 확보를 지원합니다.

    • IT-META 시스템을 기반으로 테이블 생성, 스크립트 자동 생성, 사용자 접근 이력 관리 등을 운영합니다.

2.2 MLOps 체계 구축

  • ML/DL 모델 전 생명주기 관리:

    • 데이터 준비 → 모델 개발 → 학습/테스트 → 배포 → 모니터링 → 재학습 및 업데이트까지의 전체 프로세스를 체계적으로 관리합니다.

    • ML 플랫폼 내 리스크 관련 모델(예: 기업여신, 신용평가 등)을 이관 및 운영하며, API 배포 및 파이프라인 최적화를 통해 성능 향상에 기여합니다.

  • 리스크 Agent 중심의 모델 운영:

    • 리스크 Agent를 활용한 데이터 파이프라인 마이그레이션 및 검증 후 확산을 통해 모델의 신뢰성과 정확성을 확보합니다.

    • 모델 성능을 AI 친화적인 데이터 구조와 시맨틱 데이터 기반으로 지속적으로 개선합니다.

  • MLOps 자동화 및 모니터링:

    • 모델 버전 관리, 재학습 조건 자동화, 모델 성능 추적, 모델 감시 및 경고 시스템을 구축합니다.

    • 모델의 성능 향상, 정확도, 사용률, 오류 발생 등을 대시보드로 실시간 모니터링하여 지속적인 최적화를 지원합니다.

2.3 DataOps와 MLOps 통합

  • 데이터와 모델 간 연계 강화:

    • 데이터 파이프라인과 ML/DL 모델 학습 및 배포 프로세스를 연계하여 데이터 흐름과 모델 개선을 일관되게 운영합니다.

    • 시맨틱 데이터와 메타데이터를 기반으로 데이터의 의미와 모델의 입력/출력을 연결하여 모델 성능 향상에 기여합니다.

  • 통합 운영 및 거버넌스:

    • DataOps와 MLOps의 운영 프로세스를 통합하여 데이터 품질, 모델 정확도, 사용자 접근 제어 등을 일관된 체계로 관리합니다.

    • 컬럼 단위 암호화, 마스킹, 해시 기능을 통해 데이터 보호와 모델의 안전한 활용을 지원합니다.

    • 거버넌스 정책을 기반으로 데이터 및 모델의 품질, 정합성, 규제 준수를 모니터링하며, KPI 기반 평가 및 개선 방향을 제시합니다.

이와 같이 DataOps와 MLOps 체계를 구축함으로써, KB는 데이터 기반의 AI 경쟁력을 확보하고, 미래형 디지털 업무 환경을 구축하는 데 기여할 수 있습니다.

3.3. 핵심 기술 요소

DataOps와 MLOps 체계 구축을 위한 핵심 기술 요소는 데이터 처리와 AI 모델의 전체 라이프사이클을 효율적으로 관리하고 최적화하기 위한 기술과 프로세스의 조합입니다. 아래는 참조 문서를 바탕으로 정리한 주요 기술 요소입니다:

1. DataOps 체계 구축

DataOps는 데이터 파이프라인의 자동화, 모니터링, 품질 관리를 통해 데이터의 신뢰성과 효율성을 높이는 프레임워크입니다. 핵심 기술 요소는 다음과 같습니다:

  • 표준화된 데이터 수집 체계:

    • 실시간, 배치, API를 통한 표준화된 데이터 수집

    • WiseNet, KB-Drive API 연계를 통한 비정형 데이터 처리

    • 레가시 시스템과의 호환성 유지 (한글 캐릭터셋, 데이터 형식 등)

    • 메타데이터 기반의 데이터 파이프라인 운영 관리

    • 내부 ↔ 클라우드 간 최적화된 데이터 전송 구현

  • 데이터 통합 및 저장소 구축:

    • 레이크하우스 아키텍처 구현 (Iceberg, DeltaLake 등 오픈 테이블 포맷)

    • 브론즈(Bronze) Layer: 원천 데이터 보존

    • 실버(Silver) Layer: 정제된 데이터 저장 및 처리

    • 고성능 및 확장성 기반의 클라우드 기반 저장소 (Object Storage 활용)

    • 멀티 엔진 지원 구조 (호환성 강화)

  • 데이터 품질 및 거버넌스 강화:

    • 데이터 품질 측정 기반의 데이터 제품화 (Data as a Product)

    • 데이터 카탈로그 및 메타정보 관리 확대

    • 데이터 접근 제어 및 로깅 체계 강화

    • 마스킹, 해시 등을 통한 개인정보 보호 체계 구축

    • 규제 준수 모니터링 및 접근 로깅 강화

  • 자동화 및 모니터링:

    • 데이터 파이프라인 자동화 및 실시간 모니터링

    • 데이터 이동 및 유연한 확장이 가능한 컴퓨팅/스토리지 분리 구조

    • 자원 최적화 및 자동 확장 기술 적용

    • 실시간, 정형, 비정형 데이터의 저장 및 처리

2. MLOps 체계 구축

MLOps는 AI 모델의 개발, 배포, 모니터링, 재학습을 체계적으로 관리하는 프레임워크로, AI의 안정성과 신뢰성을 높이기 위해 다음과 같은 기술 요소가 필요합니다:

  • 모델 개발 및 배포:

    • ML 플랫폼 내 리스크 관련 모델 (기업여신, 신용평가 등)

    • 리스크 Agent 기반의 데이터 및 파이프라인 마이그레이션

    • API 배포 및 파이프라인 이관

    • 리스크 Agent 중심의 데이터 전환 및 검증

  • 모델 성능 평가 및 품질 관리:

    • 생성형 AI 및 Agentic AI 기술을 활용한 모델 평가

    • BLEU, ROUGE, F1-score 등 생성 품질 평가 지표

    • Retrieval 성능 지표 (정밀도, 재현율, MRR 등)

    • Hallucination 감지 및 측정

    • 사용자 만족도 점수 및 수동 평가

  • 모델 모니터링 및 재학습:

    • 모델 성능의 실시간 모니터링 및 이상 탐지

    • 모델의 성능 저하 시 자동 롤백 기능

    • 성능 기준에 따른 재학습 및 업데이트

    • 감사 로그 및 버전 관리 체계

    • RAG 시스템의 검색 정확도 및 Grounding 일관성 모니터링

  • 모델 배포 및 프로모션 파이프라인:

    • 실험 환경 → 프로덕션 환경으로의 구조화된 이전

    • 자동화된 테스트 스위트 실행

    • 성능 기준에 따른 검증

    • 민감한 업데이트에 대한 수동 승인 절차

    • 점진적 롤아웃 및 실시간 모니터링

    • 버전 추적 및 감사 이력 관리

3. DataOps와 MLOps의 통합

DataOps와 MLOps는 데이터와 모델의 라이프사이클을 연결하는 데 중요한 역할을 합니다. 이를 통합하기 위한 핵심 기술 요소는 다음과 같습니다:

  • 모던 데이터 플랫폼 아키텍처:

    • 클라우드 기반의 레이크하우스 아키텍처

    • 다양한 유형의 데이터 통합 저장 및 처리

    • 멀티모달 데이터 지원

    • 기술 트렌드 변화 및 신규 서비스 출시 대응 체계

  • 통합된 모니터링 및 관제 시스템:

    • Kubernetes 기반의 컨테이너 오케스트레이션

    • 네임스페이스 분리 및 인프라 모니터링

    • 데이터 및 모델 활동에 대한 실시간 추적

    • 불변의 감사 로그 시스템

    • 성능 지표 및 비용 추적 시스템 (Cost Intelligence)

  • 보안 및 규제 준수:

    • 행위 기반의 접근 제어 및 감사 로그

    • API 게이트웨이를 통한 제로트러스트 보안

    • 네트워크 세분화 및 방화벽 정책

    • 데이터 암호화 (AES-256, TLS 1.3 등)

    • HSM(Hardware Security Module) 통합

  • 기술 독립성 및 출구 전략:

    • 오픈 테이블 포맷 (Iceberg, DeltaLake) 활용

    • 다양한 분석 솔루션 (오픈소스, 상용, 클라우드 서비스) 호환

    • 기술 지원 및 유지보수의 유연성 확보

4. 기대 효과

  • AI Agent 성능 향상을 통한 데이터 기반 AI 경쟁력 확보

  • 디지털 트윈, 디지털 워크플로우 등 미래형 업무 방식 구현

  • 내·외부 데이터 융합을 통한 인사이트 기반의 신규 서비스 창출

  • 모던 데이터 플랫폼 도입으로 지속적인 개선 및 디지털 프로세스 혁신 가속

  • 통합 거버넌스 체계를 기반으로 데이터 정합성 및 접근성 개선

  • 기술독립성 확보 및 유연한 출구 전략으로 유지보수 비용 절감

이러한 기술 요소를 기반으로 DataOps와 MLOps 체계를 구축하면, 데이터와 모델의 라이프사이클을 체계적으로 관리하면서도 규제 준수, 보안, 성능 향상을 동시에 달성할 수 있습니다.

10. 리스크 Agent 개발

10.1. 기술 설명

리스크 Agent는 KB가 추진 중인 AI Agent 기반의 데이터 현대화 전략의 핵심 요소로, 리스크 관리, 감사, 모니터링 등 고난이도 리스크 관련 업무를 AI 기술을 활용하여 자동화하고 최적화하기 위한 AI Agent입니다. 이 Agent는 내부 및 외부 데이터를 기반으로 리스크 판단, 예측, 대응을 수행하며, 고객 리스크, 신용 리스크, 기업여신 리스크, 시장 리스크 등 다양한 리스크 유형에 대한 실시간 분석 및 의사결정 지원을 목표로 설계되었습니다.

리스크 Agent는 ML/DL 분석 인프라와 MLOps 체계를 기반으로, 리스크 관련 데이터 수집, 처리, 모델 학습, 예측, 모니터링, 재학습 등 전 과정을 자동화합니다. 특히, 리스크 업무에서 활용되는 약 600개의 데이터 파이프라인과 데이터 웨어하우스(DW) 정보가 리스크 Agent에 이관되어, 고도화된 리스크 판단 모델을 구축하고, AI-Ready 데이터를 기반으로 리스크 예측 및 대응 전략 수립이 가능하도록 설계되었습니다.

리스크 Agent 개발의 핵심 기술 요소는 다음과 같습니다:

  1. 데이터 통합 및 가상화 기반 파이프라인 개발

    • 데이터 레이크하우스와 데이터 가상화를 기반으로, 이기종 DB와 리스크 관련 데이터를 결합하여 리스크 판단에 필요한 종합적인 데이터 집합을 구성합니다.

    • DW 데이터 수집 및 적재 파이프라인을 구축하여, 가공된 데이터를 리스크 Agent에 전달하고, 성능 최적화를 위한 데이터 복제 최소화 전략이 적용됩니다.

  2. ML/DL 분석 인프라 및 MLOps 도입

    • ML/DL 분석 인프라는 자동화된 데이터 전처리, 모델 학습, 평가, 서빙, 모니터링을 지원합니다.

    • MLOps 체계는 분석가 작성 코드의 버전 관리, Feature Store 기반의 모델 공유, 자동화된 파이프라인 운영 등을 포함하며, 리스크 모델의 품질 평가 및 재학습을 지원합니다.

    • 리스크 Agent 내 ML 모델 파이프라인은 기업여신, 신용평가 등 다양한 리스크 모델을 포함하며, API 배포 및 모델 서빙이 가능하도록 설계됩니다.

  3. 외부 데이터 연계 및 표준 수집 체계

    • 금융감독원, 금융위원회, 한국은행, Bloomberg, IEA 등 공공 및 외부 기관의 데이터가 KbaaS API Proxy를 통해 리스크 Agent에 연계됩니다.

    • 표준화된 데이터 수집 체계는 실시간 스트림즈, 배치, API 연계 등을 통해 데이터 수집 기준을 정립, 데이터 수집 작업의 소요기간을 단축하고, 데이터 품질 관리 및 오류 알림 기능을 포함합니다.

  4. 데이터 거버넌스 및 보안 강화

    • 데이터 민감도에 따른 분류 및 접근 제어 강화, 마스킹, 해시 등 개인정보 보호 체계가 구축되어, 리스크 Agent 내 데이터의 보안성과 규제 준수를 보장합니다.

    • 데이터 접근 로깅 및 모니터링 체계가 강화되어, 데이터 사용 내역 추적 및 감사 기능이 포함됩니다.

  5. 데이터 카탈로그 및 시맨틱 레이어 확대

    • 정형/비정형 데이터 카탈로그가 확대되어, 리스크 Agent 내에서 데이터 검색 및 활용이 용이하도록 지원됩니다.

    • 시맨틱 데이터 검색과 SQL 에이전트의 Text-to-SQL 연동을 통해, 사용자 쿼리를 리스크 관련 데이터로 변환 및 처리할 수 있습니다.

  6. 클라우드 인프라 최적화 및 확장성

    • 리스크 Agent는 클라우드 기반의 Modern Data Platform 아키텍처를 기반으로 구축되며, IaC 기반의 인프라 구성, 자동화된 CI/CD 파이프라인, 고가용성 및 복구 체계가 적용됩니다.

    • Kubernetes 기반의 확장성 및 자동 스케일링 기능이 포함되어, 리스크 업무 증가에 따른 성능 요구사항을 충족합니다.

리스크 Agent는 KB의 리스크 관리 전략을 디지털화하고, AI 기술을 기반으로 리스크 예측 및 대응을 실시간으로 수행함으로써, 리스크 기반의 전략적 의사결정을 지원하고, 리스크 관리 효율성과 정확성을 극대화할 수 있는 핵심적인 AI Agent로 설계되었습니다.

10.2. RFP 기반 구현 방향

  • 리스크 Agent를 위한 데이터 현대화를 수행하여 리스크 판단을 위한 데이터를 준비합니다.

    • 리스크 Agent가 정확한 판단을 위해 내·외부 데이터를 수집·처리·활용할 수 있도록 표준화된 데이터 수집 체계를 구축합니다.

    • 실시간 스트림즈, 배치, API 등을 통한 표준화된 데이터 수집 방식을 도입하고, 레이크하우스 기반의 데이터 통합 인프라를 구축합니다.

    • 비정형 데이터(문서, 영상, 음성 등)의 탐색 및 활용을 위한 데이터 카탈로그를 고도화하고, AI가 이해하고 활용할 수 있도록 시맨틱 데이터를 준비합니다.

    • 외부 데이터(금융감독원, Bloomberg 등)를 KbaaS API Proxy를 통해 수집·적재하고, 과거 데이터 이력 관리 체계를 구축합니다.

    • 리스크 Agent가 참조하는 약 600개의 DW 데이터 및 파이프라인을 이관하여 데이터 마이그레이션 및 마트 생성을 수행합니다.

  • ML/DL 분석 환경을 기반으로 리스크 Agent를 개발하고 운영합니다.

    • ML 플랫폼 내 기존 리스크 관련 모델(기업여신, 신용평가 등)을 리스크 Agent에 이관하고, 파이프라인을 재구성합니다.

    • DataOps 및 MLOps 체계를 도입하여 ML/DL 모델의 훈련, 배포, 모니터링을 자동화하고, 모델의 정확성과 신뢰성을 확보합니다.

    • AI가 이해할 수 있는 구조화된 데이터(시맨틱 데이터)를 기반으로 리스크 판단 모델을 개선하고, 데이터 기반의 인사이트를 도출합니다.

  • 리스크 Agent와 어플리케이션의 연동을 통해 리스크 판단을 서비스에 반영합니다.

    • 리스크 판단 결과를 리스크 관리 업무 어플리케이션에 실시간으로 제공하여 의사결정 지원을 강화합니다.

    • 리스크 Agent에서 생성된 보고서를 PDF로 변환하여 리스크 관리자에게 제공하고, 서비스 중심의 데이터 전환을 수행합니다.

    • 리스크 Agent와 어플리케이션 간의 데이터 흐름과 인터페이스를 표준화하고, 클라우드 네이티브 아키텍처를 기반으로 확장성을 확보합니다.

이러한 방향으로 리스크 Agent를 구현함으로써, KB는 AI 기반의 리스크 판단 능력을 강화하고, 데이터 기반의 디지털 트랜스포메이션을 가속화할 수 있습니다.

10.3. 핵심 기술 요소

  • 리스크 데이터 현대화

    • 리스크 관리에 필요한 정형/비정형 데이터를 통합하여 레이크하우스 아키텍처 기반의 데이터 인프라를 구축.

    • 브론즈, 실버, 골드 레이어로 구성된 메탈레이드 구조를 통해 데이터 품질과 접근성을 강화.

    • 네오알리의 온톨로지 기반 시맨틱 데이터 모델을 도입하여 데이터 간 의미적 관계를 정의하고, 리스크 Agent의 이해력과 판단력을 향상.

  • ML/DL 분석 환경 기반 리스크 Agent 개발

    • MLOps 기반의 ML/DL 분석 환경을 구축하여 데이터 전처리, 학습, 평가, 서빙, 모니터링까지의 자동화 파이프라인을 구현.

    • 네오알리의 온톨로지 기술을 활용하여 Agent가 자연어 쿼리나 SQL 기반의 요청을 이해하고, 의미적 맥락에 맞는 리스크 판단을 수행.

    • 시맨틱 레이어와 결합하여 Agent가 데이터의 의미를 파악하고, 비정형 데이터(문서, 이메일, 보고서 등)를 분석하여 리스크 인사이트를 도출.

  • 리스크 Agent와 어플리케이션 연동

    • 리스크 Agent는 기존 리스크 관리 어플리케이션과 시스템(예: 데이터 웨어하우스, 리스크 모델 등)과 연동하여 실시간 및 배치 방식으로 데이터를 처리.

    • API 및 이벤트 기반의 통합 방식을 통해 Agent가 다양한 시스템과 상호작용 가능.

    • 네오알리의 온톨로지 기반 연결성을 활용하여 Agent가 도메인별 데이터 구조를 이해하고, 어플리케이션 간의 데이터 흐름을 정확하게 모델링.

기술적 강점 및 차별화 포인트

  • 네오알리 온톨로지 기반 시맨틱 데이터 모델링

    • 데이터 간의 의미적 관계를 명확히 정의하여 Agent가 인간과 유사한 방식으로 데이터를 이해.

    • 예: \"신용 평가\"라는 개념은 \"대출 신청자\", \"재무 상태\", \"기존 신용 기록\", \"리스크 점수\" 등과 연결되어 있으며, 온톨로지 기반으로 이러한 관계를 정의하고 Agent가 이를 기반으로 판단.

  • MLOps 기반 리스크 분석 자동화

    • 데이터 수집 → 전처리 → 모델 학습 → 모델 서빙 → 모니터링의 전체 프로세스를 자동화하여 리스크 Agent의 판단 정확도와 신속성을 확보.

    • 현행 리스크 모델(예: 기업 여신, 신용 평가 등)과의 연동을 통해 Agent가 기존 모델을 기반으로 리스크 판단을 수행하며, 모델 재학습 및 최적화 가능.

  • Agent-Driven 리스크 파이프라인

    • 리스크 Agent가 자체적으로 데이터 수집, 분석, 보고서 생성, PDF 변환 등 엔드-투-엔드 작업을 수행.

    • 시스템 이벤트, 사용자 요청, 타이머 기반으로 Agent가 자율적으로 리스크 판단을 수행.

  • 4. 역할 분담 및 협력 방안

    • 데이터 전문 회사와 AI 전문 회사의 역할 분담, 협력 방안, 인력 구성 등.

    • 데이터 전문 회사와 AI 전문 회사의 역할 분담 및 시너지 창출을 위한 협력 방안은 다음과 같습니다.

    1. 역할 분담

      • 데이터 전문 회사:

        • 데이터 수집 및 정제: 내부/외부의 정형 및 비정형 데이터를 수집하고, 품질을 보장하는 데이터 정제 프로세스를 수행합니다.

        • 데이터 인프라 구축: 데이터 레이크하우스, 데이터 메쉬, 클라우드 기반의 데이터 저장소 구축 및 운영을 담당합니다.

        • 메타데이터 관리 및 시맨틱 데이터 구축: AI가 이해하고 활용할 수 있도록 데이터의 의미, 관계, 맥락 등을 정의하고, 시맨틱 레이어를 구축합니다.

        • 데이터 거버넌스 및 보안: 데이터 분류, 접근 제어, 로깅 및 규제 준수를 강화하여 데이터의 민감도와 보안을 관리합니다.

      • AI 전문 회사:

        • AI 모델 개발 및 최적화: 데이터 전문 회사가 제공한 AI-Ready 데이터를 기반으로 ML/DL 모델을 개발하고, 성능을 최적화합니다.

        • Agent 기반의 업무 지원 시스템 구축: 생성형 AI, Agentic AI 기술을 활용하여 업무 자동화 및 지원 시스템을 구축합니다.

        • 데이터 활용 기반의 인사이트 도출: 데이터를 분석하여 의미 있는 인사이트를 도출하고, 이를 통해 신규 서비스나 의사결정에 기여합니다.

        • 사용자 맞춤형 AI 서비스 개발: 사용자 니즈를 반영한 맞춤형 AI 서비스를 설계하고, 서비스 운영 및 피드백을 통한 지속적인 개선을 수행합니다.

    2. 협력 방안

      • 데이터-모델 피드백 루프 구축:

        • AI 전문 회사가 개발한 모델의 성능 개선을 위해 데이터 전문 회사가 제공하는 데이터 품질을 지속적으로 고도화합니다.

        • AI 모델의 결과를 다시 데이터 전문 회사로 피드백하여 데이터셋을 풍부하게 하고, 데이터 정제 과정에 반영합니다.

      • 데이터-모델 통합 플랫폼 운영:

        • 두 회사가 협력하여 클라우드 기반의 통합 데이터 플랫폼을 운영합니다. 이 플랫폼은 데이터 전문 회사가 제공하는 데이터와 AI 전문 회사가 개발한 모델을 연결하고, 실시간으로 데이터를 기반으로 AI가 활용할 수 있도록 지원합니다.

        • API, 가상화, 데이터 서빙 기술 등을 활용하여 두 회사 간의 데이터 및 모델 연결성을 강화합니다.

      • 인력 협력 및 교육:

        • 데이터 전문 회사와 AI 전문 회사는 인력 교류 및 교육 프로그램을 통해 서로의 전문성을 이해하고, 협력 역량을 향상시킵니다.

        • 데이터 과학자, AI 엔지니어, 비즈니스 분석가 등이 협력 프로젝트에 참여하여, 데이터 기반의 AI 솔루션을 공동으로 개발합니다.

      • 데이터 및 AI 기반의 서비스 혁신:

        • 두 회사가 협력하여 데이터와 AI를 결합한 신규 서비스를 개발합니다. 예를 들어, 고객 서비스를 위한 디지털 트윈, 리스크 관리를 위한 리스크 Agent, 금융 분석을 위한 지능형 분석 솔루션 등입니다.

        • AI 전문 회사가 제공하는 인사이트를 데이터 전문 회사가 데이터 인프라를 통해 지원하여, 서비스의 품질과 속도를 높입니다.

      • 데이터 및 AI 기술 표준화:

        • 두 회사가 협력하여 데이터 및 AI 기술 표준을 정립하고, 이를 통해 조직 전반의 데이터 및 AI 기술 수준을 높입니다.

        • 표준화된 데이터 포맷, AI 모델 아키텍처, 데이터 거버넌스 정책 등을 공동으로 개발하고 운영합니다.

    3. 시너지 효과

      • 데이터-모델 기반의 AI 성능 향상: 데이터 전문 회사의 고품질 데이터와 AI 전문 회사의 최적화된 모델이 결합하여, AI 성능을 극대화합니다.

      • 데이터 기반의 신규 서비스 창출: 데이터와 AI의 결합을 통해 기존 서비스를 혁신하거나, 새로운 서비스를 창출할 수 있습니다.

      • 데이터 및 AI 기술의 표준화와 확산: 두 회사의 협력은 데이터 및 AI 기술 표준화를 촉진하고, 조직 전반으로 기술을 확산시킬 수 있습니다.

      • 비용 절감과 효율성 향상: 데이터 전문 회사와 AI 전문 회사의 협력은 데이터 및 AI 기술의 중복 투자를 줄이고, 효율적인 자원 활용을 가능하게 합니다.

    이와 같은 역할 분담과 협력 방안을 통해 데이터 전문 회사와 AI 전문 회사는 서로의 강점을 결합하여, 데이터 기반의 AI 혁신을 이끌어낼 수 있습니다.

    데이터 플랫폼 및 AI Agent 구축 방향 요약 및 요건 정의

    본 프로젝트는 클라우드 네이티브 아키텍처를 기반으로, 다양한 유형의 데이터를 통합 저장하고 처리할 수 있는 데이터 레이크하우스 구조를 도입하는 것을 목표로 합니다. 이는 데이터 웨어하우스와 레이크의 장점을 결합한 형태로, 실시간 및 배치 방식으로 수집된 정형/비정형/반정형 데이터를 단일 스토리지 레이어에서 관리할 수 있는 구조입니다. 이에 따라 AI Agent가 데이터를 효과적으로 활용할 수 있도록 AI 친화적인 데이터 구조와 의미 기반 시맨틱 데이터를 구축하고, 다양한 데이터 소스와 연결성을 확보하는 방향으로 설계되었습니다.

    1. 데이터 레이크하우스 구조

    • 오픈테이블 포맷(Iceberg, Delta Lake) 적용으로 데이터의 자율성과 확장성을 확보

    • 컴퓨팅/스토리지 분리 구조를 도입하여 유연한 리소스 관리 및 비용 효율성 강화

    • 멀티엔진 지원으로 다양한 분석 엔진(대규모 데이터 처리, 실시간 분석 등)과의 호환성 확보

    • 브론즈(Bronze) → 실버(Silver) → 골드(Gold) 레이어 구조를 통해 데이터 품질 및 정제 단계를 체계적으로 관리

    2. AI Agent 지원 및 데이터 연결성

    • API, MCP, 가상화 기술 등을 활용하여 AI Agent가 다양한 데이터 소스에 연결
